# 个人博客：应用场景、项目需求、进阶方向与技术难点

本文档梳理个人博客系统在前后端分离（如 Spring Boot + Vue）场景下的典型应用、功能需求、进阶开发方向及常见技术难点，便于学习规划与项目设计参考。标准版能力就绪后的 **Spec 增量**见第九章，**公开上线优先级与检查清单**见第十章。

## 一、常见应用场景

个人博客不只是「发文章」，在不同场景下侧重点不同：


| 场景            | 典型用户     | 核心诉求               |
| ------------- | -------- | ------------------ |
| **个人技术博客**    | 开发者、学生   | 记录学习笔记、展示作品、SEO 引流 |
| **内容创作者主页**   | 自媒体、设计师  | 作品集 + 文章，偏展示与品牌    |
| **团队/公司技术博客** | 小团队、开源项目 | 文档、更新日志、社区沟通       |
| **知识库 / 笔记站** | 个人或小团队   | 分类检索、标签、长期沉淀       |
| **教学/课程配套站**  | 讲师       | 章节文章、作业、评论互动       |
| **轻量 CMS**    | 运营人员     | 后台发文、定时发布、多角色协作    |


新手练手时，通常从 **「个人技术博客」** 起步：用户少、功能可控，又能覆盖前后端分离的大部分核心能力。

## 二、项目需求（由浅入深）



### 1. 基础版（MVP，适合第一阶段）



#### 用户侧

- 文章列表（分页）
- 文章详情（标题、正文、发布时间）
- 按分类 / 标签浏览
- 简单搜索（标题关键词）



#### 管理侧

- 管理员登录
- 文章的增删改查（CRUD）
- 分类、标签管理



#### 非功能

- 前后端分离（REST API + JSON）
- 统一响应格式（如 `code / message / data`）
- 基础异常处理、参数校验

---



### 2. 标准版（接近真实博客）



#### 内容

- Markdown 或富文本编辑
- 文章草稿 / 已发布 / 已下架
- 封面图、摘要、阅读量
- 置顶、推荐文章



#### 用户与权限

- 注册 / 登录（JWT）
- 角色：访客、作者、管理员
- 仅作者可改自己的文章
- 登录用户修改本人密码（校验旧密 + BCrypt；见进阶 Spec `blog-password-change`）



#### 互动

- 评论（可先做一级评论，不做楼中楼）
- 点赞（文章 / 评论）



#### 体验

- 首页轮播或精选
- 归档（按年月）
- 站点配置（站点名、简介、社交链接）

## 三、典型功能模块拆解

```text
博客系统
├── 用户模块     注册、登录、资料、改密、角色权限
├── 文章模块     CRUD、状态、Markdown/HTML、SEO 字段
├── 分类/标签    树形分类、多对多标签
├── 评论模块     审核、回复、反垃圾
├── 媒体模块     上传、存储、引用管理
├── 站点配置     主题、导航、友链、关于页
├── 统计模块     PV/UV、热门排行
└── 系统模块     日志、备份、定时任务（定时发布）
```



### 核心表关系（概念上）

- `User` 1:N `Article`
- `Article` N:M `Tag`
- `Article` N:1 `Category`
- `Article` 1:N `Comment`
- `Comment` 可自关联（parent_id）实现回复

## 四、进阶开发方向



### 1. 架构与工程化

- 单体 → 模块化（按 domain 分包）→ 微服务（用户服务、内容服务、搜索服务）
- API 版本管理、OpenAPI/Swagger 文档
- 统一网关、配置中心（Nacos 等）



### 2. 数据与搜索

- MySQL 索引优化、慢查询分析
- 读写分离、分库分表（流量大时）
- Elasticsearch 全文搜索、搜索建议



### 3. 缓存与高并发

- Redis：热点文章、Session、计数器、分布式锁
- 缓存穿透/击穿/雪崩的处理策略
- 消息队列（RabbitMQ/Kafka）：异步通知、削峰



### 4. 安全与合规

- Spring Security + JWT 刷新令牌
- RBAC 细粒度权限（按钮级）
- 登录用户本人改密（校验旧密、BCrypt；见 `blog-password-change`）
- 内容审核、防刷、IP 限流



### 5. 前端进阶

- Vue/React 管理后台（Element Plus / Ant Design Pro）
- Markdown 编辑器（Vditor、Toast UI）
- 前台 SSR（Nuxt/Next）提升 SEO
- 主题切换、暗黑模式、响应式布局



### 6. DevOps 与可观测性

- Docker Compose 一键部署（MySQL + Redis + 后端 + Nginx + 前端）
- GitHub Actions 自动构建部署
- 日志（ELK）、链路追踪、Prometheus 监控



### 7. 产品化扩展

- Newsletter 邮件订阅
- RSS/Atom 输出
- 多租户 SaaS 博客平台
- AI 辅助：摘要生成、标签推荐、智能搜索

## 五、常见技术难点



### 1. 内容与展示


| 难点                 | 说明                                           |
| ------------------ | -------------------------------------------- |
| **Markdown 渲染与安全** | 存储 MD，展示转 HTML；需防 XSS（如禁用危险标签、用成熟 sanitizer） |
| **富文本一致性**         | 编辑器 HTML 杂乱，移动端样式难统一                         |
| **大文章性能**          | 超长文加载、目录锚点、代码高亮不要拖垮首屏                        |




### 2. 权限与认证


| 难点                | 说明                                    |
| ----------------- | ------------------------------------- |
| **JWT 无状态 vs 登出** | Token 难立即失效 → 黑名单、短过期 + Refresh Token |
| **接口权限粒度**        | 「只能改自己的文章」要在 Service 层校验，不能只靠前端       |
| **前后端角色不同步**      | 路由守卫 + 后端 `@PreAuthorize` 双重校验        |




### 3. 评论与互动


| 难点        | 说明                  |
| --------- | ------------------- |
| **楼中楼结构** | 邻接表 vs 路径枚举；分页与排序复杂 |
| **反垃圾**   | 验证码、频率限制、敏感词、人工审核队列 |
| **点赞幂等**  | 同一用户重复点赞、并发计数准确性    |




### 4. 搜索与列表


| 难点            | 说明                                          |
| ------------- | ------------------------------------------- |
| **多条件筛选**     | 分类 + 标签 + 关键词 + 时间范围，动态 SQL 或 Specification |
| **分页深翻页**     | `offset` 很大时变慢 → 游标分页、搜索引擎                  |
| **搜索与 DB 同步** | 文章更新后 ES 索引延迟、失败重试                          |




### 5. 文件与媒体


| 难点       | 说明                   |
| -------- | -------------------- |
| **上传安全** | 类型校验、大小限制、文件名与路径注入   |
| **存储方案** | 本地 vs OSS；私有桶与签名 URL |
| **引用清理** | 删文章后孤儿图片如何处理         |




### 6. 性能与缓存


| 难点        | 说明                          |
| --------- | --------------------------- |
| **缓存一致性** | 更新文章后清哪些 key、先更 DB 还是先删缓存   |
| **热点文章**  | 高并发读；缓存 + 静态化部分页面           |
| **阅读量统计** | 每次 PV 写库压力大 → Redis 累加再异步落库 |




### 7. 前后端协作


| 难点             | 说明                                          |
| -------------- | ------------------------------------------- |
| **跨域与 Cookie** | 前后端不同域时 Token 放 Header 还是 Cookie            |
| **统一错误码**      | 业务异常、校验失败、401/403 前端如何统一处理                  |
| **日期与时区**      | 后端 `LocalDateTime` 与前端展示的约定（ISO 8601 + UTC） |




### 8. 部署与 SEO


| 难点 | 说明 |
| --- | --- |
| **纯 SPA 的 SEO** | 爬虫对 JS 支持有限 → SSR、预渲染或单独落地页（见 `blog-seo`，上线后 P1） |
| **HTTPS 与混合内容** | 接口、图片资源协议一致；公网上线必上证书（第十章 P0） |
| **环境配置** | dev/test/prod 数据源、密钥分离；CORS / `BLOG_SITE_BASE_URL` / 上传目录按环境注入 |
| **反代与静态资源** | 生产需 Nginx（或同类）反代 `/api`、`/uploads`、`/feed.xml` 与 SPA history fallback；开发期 Vite proxy 不可直接当生产 |
| **持久化** | 本地上传目录需挂卷；`ddl-auto: update` 不适合长期生产 |


> 完整上线检查项与优先级见 **第十章**；可分配 Spec/Task 素材见 **第十章 §8**；对应 Spec 见 `blog-ops-docker`、`blog-prod-hardening`、`blog-rate-limit`、`blog-seo`、`blog-observability`、`blog-ops-backup`、`blog-auth-recovery`。

## 十、上线优先级与待办清单

> **结论（2026-07-15）**：标准版产品能力已超过「能给人看」的门槛，**不必等第九章剩余 Spec 全部做完再上线**。公开暴露后的卡点主要在**生产化与安全**，而非再堆评论/主题/搜索等功能。  
> **与第九章关系**：本章 §2～5 为优先级说明；**§8 为可分配 Spec 任务计划素材**（含建议目录名、卡片六要素、Task 拆解、依赖与切片）。开 Spec 时复制 §8 到 `specs/{name}/spec.md` 再补全。  
> **建议执行顺序**：先定托管方式 → `blog-prod-hardening` → `blog-ops-docker` → `blog-rate-limit` Slice-A → 冒烟 → 再考虑 P1～P3。



### 1. 优先级定义


| 级别 | 含义 | 是否阻塞首发 |
| --- | --- | --- |
| **P0** | 公开上线前宜完成；不做则易断站、裸奔或账号被撞 | 是（或强烈建议同批完成） |
| **P1** | 上线后尽快补强；有一定流量或对外分享时更明显 | 否 |
| **P2** | 有运营节奏或读压力后再做 | 否 |
| **P3** | 可选深入 / 简历加分 / 多作者协作 | 否 |



### 2. P0 — 上线前亟待完善


#### 2.1 部署与运行形态（对应 `blog-ops-docker`）

仓库目前无 Docker / Nginx 配置；开发期依赖 Vite 代理。上线至少明确：


| 项 | 现状风险 | 建议做法 | 归属 Spec |
| --- | --- | --- | --- |
| 前端生产构建与反代 | `/api`、`/uploads`、`/feed.xml` 靠本地 proxy | Nginx（或同类）反代；SPA `history` 配 `try_files` | `blog-ops-docker` |
| 站点公网 URL | `BLOG_SITE_BASE_URL` 默认 localhost | 设为 HTTPS 公网域名 | `blog-ops-docker` + hardening 文档 |
| 上传目录 | 本地 `./data/uploads` | 挂持久卷；确认公网可访问 `/uploads` | `blog-ops-docker` |
| 部署文档 | 仅有本地启动说明 | 补生产步骤或 Compose | `blog-ops-docker` |
| HTTPS | 无证书则明文传密 | 反代层终结 TLS | `blog-ops-docker` |


#### 2.2 安全与配置硬化（对应 `blog-prod-hardening` + `blog-rate-limit`）


| 项 | 默认风险 | 建议 | 归属 Spec |
| --- | --- | --- | --- |
| CORS | 仅 localhost | 生产允许真实前端 Origin | `blog-prod-hardening` |
| 管理员 / JWT / DB 密码 | 开发默认值 | 环境变量强密钥；生产拒绝默认 JWT | `blog-prod-hardening` |
| 注册是否开放 | 默认可注册 AUTHOR | 可配置关闭公开注册 | `blog-prod-hardening` |
| 库表策略 / 日志级别 | `ddl-auto: update`、DEBUG | 生产 profile：`validate` 或可控迁移 + INFO | `blog-prod-hardening` |
| 登录限流 | 仅评论有频控 | 登录失败按 IP/账号限流 | `blog-rate-limit` Slice-A |


#### 2.3 上线前产品卫生（归入 `blog-prod-hardening` 验收 Task）

- 换掉默认管理员凭据；发几篇真实内容；填好站点名 / 简介 / 关于 / 友链
- 确认评论审核策略（敏感词、待审入口）可用
- 确认封面、头像、上传图在**生产域名**下可访问
- 未合并的本地增量先稳定再部署
- **注册策略决策**：开放注册 → 后续摘取 `blog-auth-recovery`；关闭注册 → 可跳过找回


#### 2.4 P0 冒烟清单（`blog-ops-docker` / hardening 联合验收）

```text
1. HTTPS 下打开首页 / 文章列表 / 详情 / 关于
2. 管理员登录后台 → 发文 / 改站点配置 → 前台可见
3. 上传封面或图片 → 前台 URL 可打开
4. 登录用户评论 / 点赞（若开放注册）
5. /feed.xml（或约定 feed 路径）在公网域名下条目正确
6. 未登录访问管理接口返回 401；默认口令不可再用
```

##### C. `blog-rate-limit`（P0 Slice-A / P1 Slice-B）


| 项 | 内容 |
| --- | --- |
| **Problem** | 登录可被爆破；公开搜索可被刷；评论已有频控需对齐体验 |

| --- | --- | --- | --- |
| T1 | A | 登录失败计数与拒绝策略（内存即可） | 超限 429/业务码；未超限可登录 |

| T4 | B | 文档说明与评论限流对照 | 运维可知各阈值默认值 |
