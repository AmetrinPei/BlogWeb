# 个人博客：应用场景、项目需求、进阶方向与技术难点

本文档梳理个人博客系统在前后端分离（如 Spring Boot + Vue）场景下的典型应用、功能需求、进阶开发方向及常见技术难点，便于学习规划与项目设计参考。

## 一、常见应用场景

个人博客不只是「发文章」，在不同场景下侧重点不同：

|场景|典型用户|核心诉求|
|---|---|---|
|**个人技术博客**|开发者、学生|记录学习笔记、展示作品、SEO 引流|
|**内容创作者主页**|自媒体、设计师|作品集 + 文章，偏展示与品牌|
|**团队/公司技术博客**|小团队、开源项目|文档、更新日志、社区沟通|
|**知识库 / 笔记站**|个人或小团队|分类检索、标签、长期沉淀|
|**教学/课程配套站**|讲师|章节文章、作业、评论互动|
|**轻量 CMS**|运营人员|后台发文、定时发布、多角色协作|

新手练手时，通常从 **「个人技术博客」** 起步：用户少、功能可控，又能覆盖前后端分离的大部分核心能力。

## 二、项目需求（由浅入深）

### 1. 基础版（MVP，适合第一阶段）

#### 用户侧

- 文章列表（分页）
- 文章详情（标题、正文、发布时间）
- 按分类 / 标签浏览
- 简单搜索（标题关键词）

#### 管理侧

- 管理员登录
- 文章的增删改查（CRUD）
- 分类、标签管理

#### 非功能

- 前后端分离（REST API + JSON）
- 统一响应格式（如 `code / message / data`）
- 基础异常处理、参数校验

---

### 2. 标准版（接近真实博客）

#### 内容

- Markdown 或富文本编辑
- 文章草稿 / 已发布 / 已下架
- 封面图、摘要、阅读量
- 置顶、推荐文章

#### 用户与权限

- 注册 / 登录（JWT）
- 角色：访客、作者、管理员
- 仅作者可改自己的文章

#### 互动

- 评论（可先做一级评论，不做楼中楼）
- 点赞（文章 / 评论）

#### 体验

- 首页轮播或精选
- 归档（按年月）
- 站点配置（站点名、简介、社交链接）

## 三、典型功能模块拆解

```text
博客系统
├── 用户模块     注册、登录、资料、角色权限
├── 文章模块     CRUD、状态、Markdown/HTML、SEO 字段
├── 分类/标签    树形分类、多对多标签
├── 评论模块     审核、回复、反垃圾
├── 媒体模块     上传、存储、引用管理
├── 站点配置     主题、导航、友链、关于页
├── 统计模块     PV/UV、热门排行
└── 系统模块     日志、备份、定时任务（定时发布）
```

### 核心表关系（概念上）

- `User` 1:N `Article`
- `Article` N:M `Tag`
- `Article` N:1 `Category`
- `Article` 1:N `Comment`
- `Comment` 可自关联（parent_id）实现回复

## 四、进阶开发方向

### 1. 架构与工程化

- 单体 → 模块化（按 domain 分包）→ 微服务（用户服务、内容服务、搜索服务）
- API 版本管理、OpenAPI/Swagger 文档
- 统一网关、配置中心（Nacos 等）

### 2. 数据与搜索

- MySQL 索引优化、慢查询分析
- 读写分离、分库分表（流量大时）
- Elasticsearch 全文搜索、搜索建议

### 3. 缓存与高并发

- Redis：热点文章、Session、计数器、分布式锁
- 缓存穿透/击穿/雪崩的处理策略
- 消息队列（RabbitMQ/Kafka）：异步通知、削峰

### 4. 安全与合规

- Spring Security + JWT 刷新令牌
- RBAC 细粒度权限（按钮级）
- 内容审核、防刷、IP 限流

### 5. 前端进阶

- Vue/React 管理后台（Element Plus / Ant Design Pro）
- Markdown 编辑器（Vditor、Toast UI）
- 前台 SSR（Nuxt/Next）提升 SEO
- 主题切换、暗黑模式、响应式布局

### 6. DevOps 与可观测性

- Docker Compose 一键部署（MySQL + Redis + 后端 + Nginx + 前端）
- GitHub Actions 自动构建部署
- 日志（ELK）、链路追踪、Prometheus 监控

### 7. 产品化扩展

- Newsletter 邮件订阅
- RSS/Atom 输出
- 多租户 SaaS 博客平台
- AI 辅助：摘要生成、标签推荐、智能搜索

## 五、常见技术难点

### 1. 内容与展示

|难点|说明|
|---|---|
|**Markdown 渲染与安全**|存储 MD，展示转 HTML；需防 XSS（如禁用危险标签、用成熟 sanitizer）|
|**富文本一致性**|编辑器 HTML 杂乱，移动端样式难统一|
|**大文章性能**|超长文加载、目录锚点、代码高亮不要拖垮首屏|

### 2. 权限与认证

|难点|说明|
|---|---|
|**JWT 无状态 vs 登出**|Token 难立即失效 → 黑名单、短过期 + Refresh Token|
|**接口权限粒度**|「只能改自己的文章」要在 Service 层校验，不能只靠前端|
|**前后端角色不同步**|路由守卫 + 后端 `@PreAuthorize` 双重校验|

### 3. 评论与互动

|难点|说明|
|---|---|
|**楼中楼结构**|邻接表 vs 路径枚举；分页与排序复杂|
|**反垃圾**|验证码、频率限制、敏感词、人工审核队列|
|**点赞幂等**|同一用户重复点赞、并发计数准确性|

### 4. 搜索与列表

|难点|说明|
|---|---|
|**多条件筛选**|分类 + 标签 + 关键词 + 时间范围，动态 SQL 或 Specification|
|**分页深翻页**|`offset` 很大时变慢 → 游标分页、搜索引擎|
|**搜索与 DB 同步**|文章更新后 ES 索引延迟、失败重试|

### 5. 文件与媒体

|难点|说明|
|---|---|
|**上传安全**|类型校验、大小限制、文件名与路径注入|
|**存储方案**|本地 vs OSS；私有桶与签名 URL|
|**引用清理**|删文章后孤儿图片如何处理|

### 6. 性能与缓存

|难点|说明|
|---|---|
|**缓存一致性**|更新文章后清哪些 key、先更 DB 还是先删缓存|
|**热点文章**|高并发读；缓存 + 静态化部分页面|
|**阅读量统计**|每次 PV 写库压力大 → Redis 累加再异步落库|

### 7. 前后端协作

|难点|说明|
|---|---|
|**跨域与 Cookie**|前后端不同域时 Token 放 Header 还是 Cookie|
|**统一错误码**|业务异常、校验失败、401/403 前端如何统一处理|
|**日期与时区**|后端 `LocalDateTime` 与前端展示的约定（ISO 8601 + UTC）|

### 8. 部署与 SEO

|难点|说明|
|---|---|
|**纯 SPA 的 SEO**|爬虫对 JS 支持有限 → SSR、预渲染或单独落地页|
|**HTTPS 与混合内容**|接口、图片资源协议一致|
|**环境配置**|dev/test/prod 数据源、密钥分离|

## 六、建议的学习里程碑

1. **阶段 1**：文章 CRUD + 分类标签 + 前台列表详情
2. **阶段 2**：JWT 登录 + 管理后台 + Markdown 发布
3. **阶段 3**：评论 + 上传封面 + Redis 缓存首页
4. **阶段 4**：搜索（可先 MySQL LIKE，再 ES）+ Docker 部署
5. **阶段 5**：限流、监控、SEO 或微服务拆分（选做）

每完成一阶段，就有一次「可演示、可写进项目经历」的增量，比一开始堆全功能更容易坚持。

## 九、标准版完成后的 Spec 增量目录

> **用途**：标准版（内容增强、RBAC、评论点赞、站点体验）已完成后，按需摘取下表条目，各自开一份 `specs/{module}/spec.md` → `plan.md` → `tasks.md`。  
> **流程**：遵守项目 `总纲-项目技术规范文档.md` 与 `constitution.md`；重大变更禁止绕过 Spec 直接改代码。  
> **宪法注意**：当前 constitution **禁止**在无 Spec 依据时引入 Elasticsearch、Redis、消息队列、对象存储 SDK、SSR（Nuxt/Next）。表中标注「需改宪法」的条目，开 Spec 前须先修订 `constitution.md`。

### 1. 如何摘取

1. 从总览表选定 1 个 `建议 Spec 目录名`
2. 复制对应「条目卡片」中的 Problem / Metrics / AC 草稿到 `specs/{name}/spec.md`，补全六要素与 Non-Goals
3. 若依赖未完成的上游模块，先做依赖或在 Constraints 中写明假设
4. Spec ↔ Plan 迭代合格后再委派实现；PR 说明引用模块名与 Task 编号

**推荐落地顺序（可跳过）：**

```text
体验打磨 / RSS / 搜索扩字段
  → 楼中楼评论 + 审核反垃圾
    → 媒体上传 + JWT 刷新 / 限流
      → Docker 部署
        → Redis 缓存与阅读量
          → SEO 或 ES（二选一深入）
```

### 3. 条目卡片（Spec 草稿素材）

开 Spec 时可直接改写下列卡片；`Acceptance Criteria` 为建议项，落地前按可测标准裁剪。

---

#### `blog-search-enhance` — 搜索增强

| 项 | 内容 |
| --- | --- |
| **Problem** | 仅标题关键词搜索，文章多时无法按正文/摘要定位 |
| **Success Metrics** | 关键词命中摘要或正文可出现在结果中；P95 在约定数据量下可接受（如本地测 300ms 内） |
| **建议 AC** | 公开搜索支持标题 + 摘要 + 正文；空关键词行为明确；结果仍仅已发布；可选结果摘要片段 |
| **Non-Goals 建议** | 本期可不做 ES、搜索建议、相关推荐（留给 `blog-fulltext-es`） |
| **Constraints** | 优先 MySQL / JPA；引入 ES 须先改 constitution |

---

#### `blog-rss-feed` — RSS / Atom

| 项 | 内容 |
| --- | --- |
| **Problem** | 读者无法用阅读器订阅站点更新 |
| **Success Metrics** | 合法 RSS 或 Atom；订阅源含最近 N 篇已发布文章标题、链接、摘要、时间 |
| **建议 AC** | `GET` 公开 feed；仅已发布；站点名/简介取自站点配置；非法状态文章不出现 |
| **Non-Goals 建议** | Newsletter 邮件推送另开 `blog-newsletter` |
| **Constraints** | 内容编码与站点 URL 配置明确 |

---

#### `blog-reading-ux` — 阅读体验打磨

| 项 | 内容 |
| --- | --- |
| **Problem** | 长文缺少目录与稳定排版，首屏与代码块体验一般 |
| **Success Metrics** | 含多级标题的文章可生成可点击 TOC；危险 HTML 仍不可执行 |
| **建议 AC** | 详情页标题锚点 TOC；代码块高亮不拖垮首屏（可延迟加载）；列表/详情加载态（骨架或占位）；搜索无结果空态 |
| **Non-Goals 建议** | 不改文章状态机与权限模型；不做 SSR |
| **Constraints** | 以微小+局部组件为主；若新增公开 API 再升格为完整 Spec |

---

#### `blog-comment-thread` — 楼中楼评论

| 项 | 内容 |
| --- | --- |
| **Problem** | 仅一级评论，无法针对某条回复讨论 |
| **Success Metrics** | 登录用户可回复已有评论；列表能还原父子关系；深度或层数有上限且可测 |
| **建议 AC** | `parentId`（或等价）自关联；公开列表返回树或「父 + 子列表」；删除策略明确（级联或保留占位）；未登录 401 |
| **Non-Goals 建议** | 审核流、敏感词见 `blog-comment-moderation` |
| **Constraints** | 兼容现有一级评论数据；点赞目标仍含评论 |

---

#### `blog-comment-moderation` — 评论审核与反垃圾

| 项 | 内容 |
| --- | --- |
| **Problem** | 开放评论易被刷屏与垃圾内容污染 |
| **Success Metrics** | 触发敏感词或未审核策略时前台不可见或需过审；超频返回明确错误 |
| **建议 AC** | 评论状态（如待审/通过/拒绝）或敏感词拦截；ADMIN 审核接口；登录/评论接口频率限制（可与 `blog-rate-limit` 合并） |
| **Non-Goals 建议** | 不做完整风控中台；验证码可选 |
| **Constraints** | 权限在 Service 层；不在日志打评论全文中的隐私若有约定 |

---

#### `blog-media-upload` — 媒体上传

| 项 | 内容 |
| --- | --- |
| **Problem** | 封面与正文图仅能填外部 URL，无法自托管资源 |
| **Success Metrics** | 认证用户可上传允许类型图片并获得可访问 URL；超限与非法类型被拒绝 |
| **建议 AC** | 类型白名单、大小上限；返回 URL 可写入 `coverUrl` 或 MD；ADMIN/AUTHOR 权限清晰；禁止路径穿越 |
| **Non-Goals 建议** | 本期可不做视频、CDN、自动清孤儿文件（可列为后续） |
| **Constraints** | 本地磁盘可先做且不一定改宪法；OSS/MinIO SDK **须改 constitution** |

---

#### `blog-auth-refresh` — 刷新令牌与登出

| 项 | 内容 |
| --- | --- |
| **Problem** | 纯 Access JWT 难以安全登出与长时间会话 |
| **Success Metrics** | Access 短过期；Refresh 可换新 Access；登出后 Refresh 失效 |
| **建议 AC** | 登录返回 access + refresh；刷新接口；登出拉黑或删除 refresh；旧 access 过期后不可用 |
| **Non-Goals 建议** | 不做 OAuth、邮箱验证 |
| **Constraints** | 密码与 Token 全文禁止进日志；兼容现有角色声明 |

---

#### `blog-rate-limit` — 接口限流

| 项 | 内容 |
| --- | --- |
| **Problem** | 登录、评论、搜索易被刷，影响可用性 |
| **Success Metrics** | 超限返回约定错误码（如 429 或业务码）；正常流量不受影响 |
| **建议 AC** | 至少覆盖登录失败、发评论、公开搜索；按 IP 和/或 userId；配置可调 |
| **Non-Goals 建议** | 不做全站 WAF；分布式限流若用 Redis 须改 constitution |
| **Constraints** | 与统一响应格式一致 |

---

#### `blog-scheduled-publish` — 定时发布

| 项 | 内容 |
| --- | --- |
| **Problem** | 只能靠手动改状态，无法预约上线 |
| **Success Metrics** | 到点后草稿自动变为已发布且公开可见 |
| **建议 AC** | 可设置未来 `publishedAt` + 调度；错过补跑策略明确；公开列表仍只显示已发布且已到点 |
| **Non-Goals 建议** | 不做复杂工作流引擎 |
| **Constraints** | 与现有 `DRAFT/PUBLISHED/OFFLINE` 状态机兼容 |

---

#### `blog-user-profile` — 用户资料

| 项 | 内容 |
| --- | --- |
| **Problem** | 用户仅有用户名等最小字段，缺少对外展示资料 |
| **Success Metrics** | 登录用户可更新简介等资料；评论/文章侧可展示公开字段 |
| **建议 AC** | 资料读写接口；字段长度校验；头像以 URL 或依赖 `blog-media-upload` |
| **Non-Goals 建议** | 不做社交关注、私信 |
| **Constraints** | 不泄露邮箱等未约定敏感字段 |

---

#### `blog-theme-switch` — 主题切换

| 项 | 内容 |
| --- | --- |
| **Problem** | 访客端仅单一视觉主题 |
| **Success Metrics** | 用户可切换亮/暗（或预设主题）且刷新后保持 |
| **建议 AC** | 前台切换控件；偏好持久化（localStorage 即可）；关键对比度可读 |
| **Non-Goals 建议** | 不做可在线编辑的完整主题商店 |
| **Constraints** | 多为前端微小/局部变更；若站点配置下发默认主题再写 API AC |

---

#### `blog-friend-links` — 友链与关于页

| 项 | 内容 |
| --- | --- |
| **Problem** | 友链与关于内容难以后台维护 |
| **Success Metrics** | ADMIN 可维护友链列表；关于页内容可配置并前台展示 |
| **建议 AC** | 友链 CRUD 或站点配置扩展；公开只读接口；排序字段 |
| **Non-Goals 建议** | 不做友链申请审核工作流（可另开） |
| **Constraints** | 可扩展现有站点配置单例或独立表 |

#### `blog-ops-docker` — Docker 部署

| 项 | 内容 |
| --- | --- |
| **Problem** | 本地多进程启动成本高，演示与交付不稳定 |
| **Success Metrics** | 文档中一条命令可拉起依赖与应用；健康检查通过即可访问前后端 |
| **建议 AC** | `docker-compose` 含 MySQL、后端、前端（或 Nginx）；环境变量注入密钥；README 启动步骤可复现 |
| **Non-Goals 建议** | 本期可不含 K8s、多环境 GitOps |
| **Constraints** | 不把真实生产密钥写入仓库 |

---

#### `blog-redis-cache` — Redis 缓存与计数

| 项 | 内容 |
| --- | --- |
| **Problem** | 热点文章与每次详情写库阅读量，读多时压力大、一致性需策略 |
| **Success Metrics** | 首页精选/热点命中缓存；阅读量 Redis 累加并定期落库；更新文章后缓存失效可验证 |
| **建议 AC** | 约定 cache key 与 TTL；穿透/击穿基本策略；落库任务或阈值可测 |
| **Non-Goals 建议** | 不做全站 Session 迁 Redis（除非需要） |
| **Constraints** | **必须先改 constitution**；明确与 JPA 的一致性策略 |

#### `blog-seo` — SEO 基础

| 项 | 内容 |
| --- | --- |
| **Problem** | 纯 SPA 对搜索引擎与社交分享不友好 |
| **Success Metrics** | 可抓取 sitemap；文章页具备基本 meta / Open Graph |
| **建议 AC** | `sitemap.xml`；详情 OG 标题摘要封面；可选预渲染关键路由 |
| **Non-Goals 建议** | 完整 Nuxt SSR 可列为二期并改 constitution |
| **Constraints** | SSR 框架引入须改 constitution |

#### `blog-fulltext-es` — Elasticsearch 全文检索

| 项 | 内容 |
| --- | --- |
| **Problem** | 文章量大后 DB LIKE 无法满足相关性与性能 |
| **Success Metrics** | 标题+正文检索；可高亮；P95 达标；文章变更后索引最终一致 |
| **建议 AC** | 索引同步（保存/删除）；公开搜索切 ES；失败重试或补偿策略 |
| **Non-Goals 建议** | 可不做向量检索；相关推荐可作 AC 可选项 |
| **Constraints** | **必须先改 constitution**；建议在 `blog-search-enhance` 之后 |

#### `blog-observability` — 可观测性

| 项 | 内容 |
| --- | --- |
| **Problem** | 故障时缺少请求日志与健康信号 |
| **Success Metrics** | 健康检查可用；关键请求可追踪到用户/路径（无敏感信息） |
| **建议 AC** | `/actuator/health` 或等价；访问日志字段约定；可选简单指标 |
| **Non-Goals 建议** | 完整 ELK/链路追踪可二期 |
| **Constraints** | 禁止日志输出密码、Token 全文、连接串密钥 |
