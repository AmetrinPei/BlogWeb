# 上线检查清单（hardening + ops）

> 对应需求文档第十章 §2.3～2.4。  
> 应用配置：`blog-prod-hardening`；反代 / HTTPS / Compose：`blog-ops-docker`（见 [`部署方式.md`](./部署方式.md)）。  
> 环境变量约定见仓库根目录 `.env.example` / `deploy/.env.example`。

---

## 1. 生产配置（应用侧）

- [ ] 使用 `SPRING_PROFILES_ACTIVE=prod`（或等价）启动
- [ ] 已设置强密钥：`JWT_SECRET`（≥32，非开发默认串）、`DB_PASSWORD`（非 `root`）、`ADMIN_PASSWORD`（非 `admin123`）
- [ ] `BLOG_SITE_BASE_URL` 为公网 HTTPS 根（不含 localhost）
- [ ] 按需设置 `BLOG_CORS_ALLOWED_ORIGIN_PATTERNS`（前后端分域时必填；同域反代可留空）
- [ ] 已决定注册策略：`BLOG_PUBLIC_REGISTRATION_ENABLED=false`（关闭，可跳过找回）或 `true`（开放 → 上线后摘取 `blog-auth-recovery`）
- [ ] 登录失败限流已启用（默认 15 分钟内 5 次；可用 `BLOG_LOGIN_MAX_FAILURES` / `BLOG_LOGIN_WINDOW_SECONDS` 调整）
- [ ] 生产启动成功；用开发默认 JWT / `admin123` / `root` / localhost 基址时**不能**启动（自测一次失败路径）
- [ ] 生产日志为 INFO；`ddl-auto` 为 `validate`（库结构已事先对齐）

## 2. 产品卫生（§2.3）

- [ ] 默认管理员凭据已更换（配置 + 库中口令；确认 `admin` / `admin123` 不可登录）
- [ ] 已发若干真实内容（非仅种子演示）
- [ ] 站点名 / 简介 / 关于 / 友链已填写
- [ ] 评论审核策略可用（敏感词、待审入口）
- [ ] 封面、头像、上传图在**生产域名**下可访问（ops：`/uploads` 反代与卷）
- [ ] 未合并的本地实验分支未直接部署到公网

## 3. 冒烟（§2.4，与 ops 联合）

- [ ] （ops）HTTPS 下打开首页 / 文章列表 / 详情 / 关于
- [ ] 管理员登录后台 → 发文 / 改站点配置 → 前台可见
- [ ] （ops）上传封面或图片 → 前台 URL 可打开
- [ ] 登录用户评论 / 点赞（若开放注册）
- [ ] （ops）`/feed.xml` 在公网域名下条目与链接正确（依赖 `BLOG_SITE_BASE_URL`）
- [ ] 未登录访问管理写接口返回 401（或统一业务未认证）
- [ ] 默认口令不可再用

## 4. 自动化抽检（可选）

```text
cd backend
.\mvnw.cmd "-Dtest=ProdHardeningTests,ProdHardeningRegistrationClosedTests" test

# 后端已启动（开发 profile）时：
node scripts/acceptance-prod-hardening.mjs
node scripts/acceptance-login-rate-limit.mjs

# 部署配置静态检查（+ 可选 OPS_BASE 冒烟）：
node scripts/acceptance-ops-docker.mjs
```
