# 生产部署指南（轻量云 + 宝塔）

面向「想省事、尽快上线、费用别太高」的场景：一台轻量服务器装宝塔，**MySQL + 后端 + 前端同机部署**，用 Nginx 统一对外。

本地开发请看 [启动方式.md](./启动方式.md)。

## 架构说明

```text
用户 → Nginx（80/443）
         ├─ /          → 前端 dist（静态）
         ├─ /api       → Spring Boot :8080
         ├─ /uploads   → Spring Boot :8080
         └─ /feed.xml  → Spring Boot :8080
MySQL 本机 :3306（不对公网开放）
上传文件目录：/www/wwwroot/blog/data/uploads
```

前端默认请求相对路径 `/api`（见 `frontend/src/api/http.js`），与 Nginx 同域反代时**不必**设置 `VITE_API_BASE_URL`，也避开跨域问题。

| 项 | 建议 |
| --- | --- |
| 服务器 | 腾讯云 / 阿里云「轻量应用服务器」，Ubuntu 22.04，约 1 核 2G |
| 费用 | 云主机约 40–120 元/年；域名可选约 10–60 元/年；SSL 免费 |
| 域名 | 有更好；没有可先用公网 IP 访问 |
| 备案 | 国内服务器用域名开 80/443 通常需 ICP 备案；不想等可用 IP 或海外机 |

---

## 1. 买服务器并放行端口

1. 购买轻量应用服务器，系统选 **Ubuntu 22.04 纯净版**（不要选其它应用镜像，避免和宝塔冲突）。
2. 在云厂商安全组/防火墙放行：**22、80、443**；面板端口（默认 8888）建议仅限自己 IP。
3. **不要**对公网开放 `8080`、`3306`。

记下公网 IP，用 SSH 或网页终端登录。

---

## 2. 安装宝塔与运行环境

1. 打开 [宝塔官网](https://www.bt.cn)，复制 Ubuntu 一键安装命令并执行。
2. 保存输出的面板地址、用户名、密码，浏览器登录面板。
3. 在「软件商店」安装：

| 软件 | 用途 |
| --- | --- |
| Nginx | 静态站点 + 反代 |
| MySQL 8.x | 数据库 |
| JDK 21 / Java 项目管理器 | 跑 Spring Boot |
| Node.js 18 或 20 LTS | 构建前端 |

---

## 3. 创建数据库

在宝塔「数据库」：

1. 新建数据库：`blog`
2. 记下 root 密码，或新建专用用户并授权 `blog`（当前后端配置默认用户名为 `root`，最省事可直接用 root + 强密码）
3. 确认 MySQL **未**对公网监听（仅本机）

后端库相关配置（`backend/src/main/resources/application.yml`）：

- 库名：`blog`
- 用户：`root`（写死在配置里）
- 密码：环境变量 `DB_PASSWORD`（默认本地是 `root`，线上务必覆盖）

---

## 4. 上传代码并建目录

推荐目录：

```text
/www/wwwroot/blog/
  ├── backend/
  ├── frontend/
  └── data/uploads/
```

上传方式任选：

- 宝塔文件管理上传 zip 后解压
- 服务器上 `git clone` 本仓库

创建上传目录并保证 Java 运行用户可写：

```bash
mkdir -p /www/wwwroot/blog/data/uploads
```

---

## 5. 打包并启动后端

```bash
cd /www/wwwroot/blog/backend
chmod +x mvnw
./mvnw -DskipTests package
```

用环境变量启动（**线上不要用默认弱密码/密钥**）：

```bash
export DB_PASSWORD='换成强数据库密码'
export JWT_SECRET='至少32位随机字符串'
export ADMIN_PASSWORD='换成强管理员密码'
export BLOG_SITE_BASE_URL='https://你的域名'
# 若暂无域名，可先：http://公网IP
export BLOG_UPLOAD_DIR='/www/wwwroot/blog/data/uploads'

java -jar target/*.jar
```

本机自检：

```bash
curl -s http://127.0.0.1:8080/api/site
```

有 JSON 返回即后端正常。然后用宝塔「进程守护管理器」或「Java 项目」做成**开机自启**，避免 SSH 断开后进程退出。

| 环境变量 | 含义 |
| --- | --- |
| `DB_PASSWORD` | MySQL 密码 |
| `JWT_SECRET` | JWT 签名密钥（≥32 字符） |
| `ADMIN_PASSWORD` | 后台管理员密码 |
| `BLOG_SITE_BASE_URL` | 站点对外根地址（含协议） |
| `BLOG_UPLOAD_DIR` | 上传文件绝对路径（必须持久化） |

---

## 6. 构建前端

```bash
cd /www/wwwroot/blog/frontend
npm install
npm run build
```

产物在 `frontend/dist/`。  
**不要**设置 `VITE_API_BASE_URL`，保持默认 `/api`。

---

## 7. 配置 Nginx 站点

宝塔 → 网站 → 添加站点：

- 域名：填域名；暂无域名可先填公网 IP 或用默认站点
- 根目录：`/www/wwwroot/blog/frontend/dist`

在站点「配置文件」中加入（或合并进已有 `server` 块）：

```nginx
location / {
  try_files $uri $uri/ /index.html;
}

location /api/ {
  proxy_pass http://127.0.0.1:8080/api/;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
  client_max_body_size 10m;
}

location /uploads/ {
  proxy_pass http://127.0.0.1:8080/uploads/;
  proxy_set_header Host $host;
}

location = /feed.xml {
  proxy_pass http://127.0.0.1:8080/feed.xml;
}
```

说明：

- `try_files`：保证 Vue Router 刷新子路由（如 `/articles`）不 404
- `/api`、`/uploads`、`/feed.xml`：与开发态 Vite 代理一致（见 `frontend/vite.config.js`）
- `client_max_body_size`：匹配后端约 8MB 上传上限

保存后重载 Nginx。

---

## 8. HTTPS（有域名时）

1. 域名 A 记录指向服务器公网 IP。
2. 宝塔站点 → SSL → Let’s Encrypt 申请证书，开启「强制 HTTPS」。
3. 将 `BLOG_SITE_BASE_URL` 改为 `https://你的域名`，并重启后端。

## 9. 上线检查清单

- [ ] 浏览器打开首页正常
- [ ] `/api/site` 能返回站点信息（反代正常）
- [ ] 文章列表、详情正常
- [ ] 注册 / 登录 / 后台登录正常
- [ ] 上传图片后刷新仍能显示（`BLOG_UPLOAD_DIR` 持久化正常）
- [ ] 刷新 `/articles` 等子路由不 404
- [ ] `/feed.xml` 可访问
- [ ] 安全组仅开放必要端口；8080、3306 不对公网
- [ ] 已修改 `DB_PASSWORD`、`JWT_SECRET`、`ADMIN_PASSWORD`

后台账号：

- 用户名：`admin`
- 密码：以你设置的 `ADMIN_PASSWORD` 为准（未改环境变量时本地默认是 `admin123`）

## 10. 日常更新

```text
1. 拉代码或重新上传
2. 后端：cd backend && ./mvnw -DskipTests package → 重启 Java 进程
3. 前端：cd frontend && npm install && npm run build（覆盖 dist，一般不用改 Nginx）
```

定期在宝塔备份：

- MySQL 的 `blog` 库
- `/www/wwwroot/blog/data/uploads` 目录

## 常见问题

| 现象 | 可能原因 |
| --- | --- |
| 页面能开，接口全挂 | 后端未启动，或 Nginx 未反代 `/api` |
| 刷新子路由 404 | 缺少 `try_files ... /index.html` |
| 图片上传后丢失或打不开 | 未配置 `BLOG_UPLOAD_DIR`，或未反代 `/uploads` |
| RSS 打不开 | 未反代 `/feed.xml` |
| 能调通本地后端但公网失败 | 误把前端指到 `localhost:8080`，应走同源 `/api` |
| 登录态 / 管理异常 | `JWT_SECRET` 被改后旧 token 失效，需重新登录 |

---

## 安全提醒

- 线上务必更换 `JWT_SECRET`、`ADMIN_PASSWORD`、数据库密码。
- 宝塔面板改为强密码，并限制访问 IP。
- 上传目录与数据库做好定期备份。
- 切勿把含密钥的 `.env`、服务器密码提交进 Git。
