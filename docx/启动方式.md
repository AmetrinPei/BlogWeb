这是前后端分离项目，需要 **MySQL + 后端 + 前端** 都跑起来，浏览器访问前端地址。

## 1. 前置条件

- **MySQL 8.x**（本地 `127.0.0.1:3306`）
- **Java 21**
- **Node.js**（前端用）

数据库默认配置（`backend/src/main/resources/application.yml`）：

- 库名：`blog`（一般会自动创建）
- 用户：`root`
- 密码：`root`（可用环境变量 `DB_PASSWORD` 覆盖）

## 2. 启动后端


常见情况是后台进程还在占 8080。在任意 PowerShell 里这样关：

1. 查出占用 8080 的进程

netstat -ano | findstr :8080
看最后一列 PID（数字）。

2. 结束该进程

taskkill /PID <上面的PID> /F
例如 PID 是 12345：


taskkill /PID 13052 /F

一条命令直接关（连父进程一起杀）

Get-NetTCPConnection -LocalPort 8080 -ErrorAction SilentlyContinue |
  Select-Object -ExpandProperty OwningProcess -Unique |
  ForEach-Object { Stop-Process -Id $_ -Force }

开一个终端：

```powershell
cd "d:\workspace\task\mini program\backend"
.\mvnw.cmd spring-boot:run
```

成功后后端在：**http://localhost:8080**

## 3. 启动前端

再开一个终端：

```powershell
cd "d:\workspace\task\mini program\frontend"
npm install
npm run dev
```

成功后前端在：**http://localhost:5173**  
开发时 `/api` 会代理到后端 `8080`。

## 4. 怎么访问网页

| 页面 | 地址 |
| --- | --- |
| 博客首页 | http://localhost:5173/ |
| 文章列表 | http://localhost:5173/articles |
| 关于我 | http://localhost:5173/about |
| 访客登录 | http://localhost:5173/login |
| 注册 | http://localhost:5173/register |
| 我的资料 | http://localhost:5173/profile |
| 后台登录 | http://localhost:5173/admin/login |
| 后台（登录后） | http://localhost:5173/admin |

默认管理员账号：

- 用户名：`admin`
- 密码：`admin123`

## 注意

- 只开前端、不开后端时，页面能打开，但文章等接口会失败。
- 在浏览器里访问的是 **5173**，不是 8080。

## 5. 生产（简要）

公网上线请使用 **`prod` profile**，并通过环境变量注入密钥（勿把真实密钥写入仓库）：

- 参考仓库根目录 [`.env.example`](../.env.example)
- 检查项见 [`docx/上线检查清单.md`](./上线检查清单.md)
- 示例（PowerShell，按需改值）：

```powershell
$env:SPRING_PROFILES_ACTIVE = "prod"
$env:JWT_SECRET = "<至少32字符的强随机串>"
$env:DB_PASSWORD = "<强数据库密码>"
$env:ADMIN_PASSWORD = "<强管理员密码>"
$env:BLOG_SITE_BASE_URL = "https://your.domain"
# 可选：$env:BLOG_CORS_ALLOWED_ORIGIN_PATTERNS = "https://your.domain"
# 可选：$env:BLOG_PUBLIC_REGISTRATION_ENABLED = "false"
cd "d:\workspace\task\mini program\backend"
.\mvnw.cmd spring-boot:run
```

反代 / HTTPS / Docker 见 [`docx/部署方式.md`](./部署方式.md)（Spec `blog-ops-docker`）。
