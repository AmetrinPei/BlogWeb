# 生产部署（blog-ops-docker）

> 拓扑：Docker Compose = MySQL + 后端（Spring `prod`）+ Nginx（前端静态 + 反代 + HTTPS）  
> 配置硬化见 [`上线检查清单.md`](./上线检查清单.md) 与仓库 [`.env.example`](../.env.example)。  
> 本地开发仍用 [`启动方式.md`](./启动方式.md)。

---

## 1. 前置

- Docker Desktop / Docker Engine + Compose v2
- 已填写密钥（勿提交真实 `.env`）
- 证书：`deploy/nginx/ssl/fullchain.pem` + `privkey.pem`

---

## 2. 准备环境变量

```powershell
cd "d:\workspace\task\mini program"
copy .env.example deploy\.env
# 编辑 deploy\.env：填强 JWT_SECRET / DB_PASSWORD / ADMIN_PASSWORD
# BLOG_SITE_BASE_URL 须为公网 https 域名（prod 拒绝 localhost）
# 本地 Compose 冒烟可用例如：BLOG_SITE_BASE_URL=https://blog.example.com
```

关键变量与 hardening 一致：`JWT_SECRET`、`DB_PASSWORD`、`ADMIN_PASSWORD`、`BLOG_SITE_BASE_URL`。同域反代时 `BLOG_CORS_ALLOWED_ORIGIN_PATTERNS` 可留空。

可选：

| 变量 | 默认 | 说明 |
| --- | --- | --- |
| `HTTP_PORT` / `HTTPS_PORT` | 80 / 443 | 宿主机映射 |
| `SPRING_JPA_HIBERNATE_DDL_AUTO` | `update` | 首启友好；库稳定后改为 `validate` |
| `BLOG_PUBLIC_REGISTRATION_ENABLED` | `false` | 生产默认关注册 |

## 3. 证书

### 3.1 本地自签（冒烟）

```powershell
cd "d:\workspace\task\mini program\deploy\scripts"
.\gen-dev-certs.ps1
```

浏览器访问 `https://localhost` 会有自签警告，属预期。

### 3.2 公网真证

将 Let's Encrypt（或云厂商）证书复制为：

- `deploy/nginx/ssl/fullchain.pem`
- `deploy/nginx/ssl/privkey.pem`

然后重启 nginx 服务。

## 4. 构建并启动

在仓库根目录：

```powershell
cd "d:\workspace\task\mini program"
docker compose -f deploy/docker-compose.yml --env-file deploy/.env up -d --build
```

查看状态：

```powershell
docker compose -f deploy/docker-compose.yml ps
docker compose -f deploy/docker-compose.yml logs -f backend
```

停止：

```powershell
docker compose -f deploy/docker-compose.yml --env-file deploy/.env down
```

（加 `-v` 会删除 MySQL / 上传卷，慎用。）

## 5. 反代路径（Nginx）

| 路径 | 行为 |
| --- | --- |
| `/api/*` | 反代到后端 |
| `/uploads/*` | 反代到后端（卷持久化） |
| `/feed.xml` | 反代到后端 RSS |
| 其它 | SPA `try_files` → `index.html` |
| 80 | 跳转 HTTPS 443 |

## 6. 不使用 Docker 时（等价）

1. `cd backend && .\mvnw.cmd -DskipTests package` → 得到 jar  
2. `cd frontend && npm ci && npm run build` → `frontend/dist`  
3. 用本仓库 `deploy/nginx/default.conf` 思路配置主机 Nginx：静态根指向 `dist`，反代同上，TLS 在边缘终结  
4. 后端：`SPRING_PROFILES_ACTIVE=prod` + hardening 环境变量；`BLOG_UPLOAD_DIR` 指向持久目录

## 7. 冒烟（对齐第十章 §2.4）

- [ ] HTTPS 打开首页 / 文章列表 / 详情 / 关于  
- [ ] 管理员登录 → 发文 / 改站点 → 前台可见  
- [ ] 上传图片 → `/uploads/...` 可打开；重启 compose 后仍在  
- [ ] `/feed.xml` 条目链接基于 `BLOG_SITE_BASE_URL`  
- [ ] 未登录管理写接口未认证；默认口令不可用  

静态检查（无需起栈）：

```powershell
node scripts/acceptance-ops-docker.mjs
```

栈已启动且为本机自签时：

```powershell
$env:NODE_TLS_REJECT_UNAUTHORIZED = "0"
$env:OPS_BASE = "https://localhost"
node scripts/acceptance-ops-docker.mjs
```
