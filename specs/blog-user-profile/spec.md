# Feature: 用户资料与访客端账号会话

> 状态：Implemented  
> 关联：docx/个人博客-应用场景需求与进阶开发.md §九 `blog-user-profile`；constitution.md  
> 依赖：specs/blog-auth-rbac（注册 / 登录 / JWT / ADMIN·AUTHOR）；可选复用 specs/blog-media-upload（头像 URL）  
> 最后更新：2026-07-14

---

## 1. Problem Statement

标准版 `blog-auth-rbac` 已提供注册、统一登录与角色，但用户仅有用户名等最小字段，缺少可对外展示的简介与头像；同时访客端壳层几乎无账号入口——登录页复用「管理登录」且成功后默认进后台，页头无登录/注册与会话菜单。读者为评论而登录后难以回到浏览上下文；已登录的作者/管理员也不能从访客站便捷进入后台或登出/切换账号。

本期交付**用户资料 + 访客端账号会话**：登录用户可读写公开资料（展示名、简介、头像 URL）；评论/文章侧展示公开字段；访客端提供登录/注册入口与已登录菜单（资料、回后台、登出/切换账号）；从访客场景登录后可回到原浏览页。不做社交关注、私信、OAuth、邮箱验证；服务端 Refresh 失效留给 `blog-auth-refresh`。

---

## 2. Success Metrics

| 指标 | 目标值 | 度量方式 |
| --- | --- | --- |
| 资料可读写 | 登录用户可读取并更新本人公开资料；字段超限被拒绝 | API：GET/PUT 本人资料；非法长度断言业务错误 |
| 公开可展示 | 评论区与文章作者侧可见约定公开字段（非敏感） | 更新资料后公开评论/文章响应含新展示字段 |
| 访客可登录注册 | 未登录时访客端壳层可进入登录与注册 | 访客页可见入口；完成登录/注册获得会话 |
| 登录回浏览 | 从访客场景登录后可回到原公开页（非强制进后台） | 带 `redirect` 的公开路径登录后落到该路径 |
| 已登录可管会话 | 已登录用户可从访客端进入后台（有权限时）、登出或切换账号 | 菜单操作后路由与本地会话状态正确 |
| 敏感不泄露 | 资料接口与公开展示不返回密码哈希等敏感字段 | 响应断言无 password / hash |

---

## 3. User Stories

- 作为登录用户，我希望编辑展示名、简介与头像，以便在评论与文章侧有可识别的对外形象
- 作为访客，我希望在访客站一眼看到登录/注册入口，以便为评论等能力完成账号流程，而不必先找到「管理登录」
- 作为刚登录的读者，我希望登录后回到刚才阅读的文章页，以便继续评论，而不是被丢进后台
- 作为已登录的作者/管理员，我希望从访客站一键进入管理后台，以便写稿或运营
- 作为已登录用户，我希望从访客站登出或切换账号，以便在公共设备上结束会话或换号
- 作为访客，我不希望在公开接口中看到他人的密码或其它未约定敏感字段

## 4. Acceptance Criteria

### 4.1 用户资料

- [x] AC-1: 用户模型在现有 `username` / `role` 之外支持至少以下**可编辑公开资料**（Plan 固定字段名）：**展示名**（可与登录名不同）、**简介**、**头像 URL**（字符串；可空表示使用默认头像）
- [x] AC-2: 提供**需登录**的「读取本人资料」与「更新本人资料」接口；成功时统一 JSON `{ code, message, data }`；`data` 至少含 `userId`、`username`、`role` 及公开资料字段；**不得**返回 `password` / `passwordHash`
- [x] AC-3: 更新时做字段校验：展示名、简介、头像 URL 有明确长度上限（Plan 给出建议默认值）；非法或超限返回明确业务错误（非 5xx）；用户名（登录名）本期**不可**通过资料接口修改
- [x] AC-4: **头像**：允许填入可访问的图片 URL（外链或本站上传 URL）；本期**不强制**专用头像上传接口；若使用本站图，可复用 `blog-media-upload` 返回的 URL（权限仍遵循上传 Spec：仅 ADMIN/AUTHOR 可上传）
- [x] AC-5: **公开展示**：文章作者信息与评论作者信息在现有 `authorName` / `username` 基础上，至少额外返回可展示的公开字段子集（建议：展示名优先于登录名展示、头像 URL；简介可选）；无展示名时回退为 `username`；无头像 URL 时前端可用默认占位
- [x] AC-6: 未登录调用本人资料读写返回 **401**；用户只能改自己的资料（无「改他人资料」管理接口本期不做）；权限在 **Service 层**校验

### 4.2 访客端登录 / 注册与会话

- [x] AC-7: 访客端壳层（如站点 Header）在**未登录**时提供可达的**登录**与**注册**入口；注册能力复用既有 `blog-auth-rbac` 注册接口与默认角色 AUTHOR，不新建注册协议
- [x] AC-8: 访客场景下的登录页文案与心智为「账号登录」（面向读者/作者），**不得**仅呈现「管理登录」且把所有成功登录都导向后台；仍可保留 `/admin/login` 作为进入后台前的登录入口（可与统一登录页共用实现，但 redirect 规则须区分）
- [x] AC-9: **登录后跳转**：
  - 若来自访客场景且 `redirect`（或等价）指向**公开区**合法路径，登录成功后落到该路径；
  - 若来自管理场景或 `redirect` 指向 `/admin/**`，落到对应管理路径（无权限则按现有路由守卫处理）；
  - 无 `redirect` 时：访客登录默认回访客首页（或来源公开页），**不**默认进 `/admin`
- [x] AC-10: 评论等「请先登录」引导须带上返回当前页的能力（如 `redirect` 到当前文章详情），登录后可回到该页继续操作
- [x] AC-11: 访客端壳层在**已登录**时展示当前账号入口（至少可见用户名或展示名/头像），并提供菜单或等价交互，至少包含：
  - **编辑资料 / 我的资料**（可达资料编辑页或面板）；
  - **进入管理后台**：对具备后台访问权限的角色（现网为 **ADMIN** 与 **AUTHOR**）可见并可进入既有 `/admin` 区；无后台权限的角色（若未来扩展）不展示或进入后按守卫拒绝；
  - **退出登录**：清除前端会话（Token 与本地用户信息），回到未登录访客态；
  - **切换账号**：等价于退出后进入登录页（可预填空账号），以便换号登录
- [x] AC-12: 退出登录后，需登录的写操作（评论、点赞等）再次返回 **401** 或前端引导登录；本地不得残留可用 Token
- [x] AC-13: 站点品牌头像（关于页/页头站点作者形象）与**当前登录用户**头像入口职责分离，避免未登录时把站点头像误当作「我的账号」
- [x] AC-14: 关键路径具备可自动化验收手段（测试或脚本）：至少覆盖本人资料读写与校验、公开评论/文章含展示字段、未登录 401、敏感字段不出现在响应中；前端关键路径可用手工验收清单补充（登录回跳、菜单进后台/登出/切换账号）

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| 社交关注、粉丝、私信 | 卡片明确排除 |
| OAuth、第三方登录、邮箱验证、找回密码 | 留给后续；本期沿用用户名+密码 |
| Refresh Token、服务端登出拉黑 / 吊销 | 见 `blog-auth-refresh`；本期登出以清除前端会话为主 |
| 登录失败 / 全站接口限流中台 | 见 `blog-rate-limit` |
| 管理员代改任意用户资料、用户列表 CRUD 后台 | 超出本期；仅本人资料 |
| 修改登录用户名、改密独立流程 | 改密见 `blog-password-change`；本期聚焦公开资料与会话入口 |
| 新增独立 READER 角色或开放匿名评论 | 注册默认仍为 AUTHOR（与 auth-rbac 一致） |
| 强制头像必须本站上传、媒体库选头像产品化 | URL 即可；上传复用 media-upload |
| 主题切换、友链关于页配置 | 见 `blog-theme-switch` / `blog-friend-links` |

---

## 6. Constraints

- 遵守 `constitution.md`：不引入 Elasticsearch、Redis、消息队列、对象存储 SDK、SSR；业务规则在 Service 层强制执行
- 统一响应格式 `{ code, message, data }`；密码与 Token 全文禁止进日志
- **不泄露**未约定敏感字段（密码哈希、未来若有邮箱等）；公开 API 只返回约定公开资料子集
- 兼容既有注册/登录接口与 JWT claims（`userId`、`username`、`role`）；不破坏 ADMIN 种子账号与管理端路由守卫
- 头像若走本站上传，遵守 `blog-media-upload` 的类型/大小/权限；外链头像不做内容抓取校验（仅 URL 长度与基本格式）
- 关键路径验收与 PR 说明引用 `blog-user-profile` 与 Task 编号

## 7. 附录

### 7.1 术语表

| 术语 | 说明 |
| --- | --- |
| 公开资料 | 可对外展示的字段：展示名、简介、头像 URL 等；不含密码与未约定隐私字段 |
| 展示名 | 评论/文章侧优先展示的名称；可与登录 `username` 不同 |
| 访客端壳层 | 公开区布局中的全局导航/页头等，非管理后台布局 |
| 访客场景登录 | 从公开区发起的登录/注册，成功后默认留在公开区或 `redirect` 目标 |
| 管理场景登录 | 为进入 `/admin/**` 发起的登录，成功后进入后台 |
| 切换账号 | 清除当前会话并进入登录页，供另一组凭据登录 |
| 进入管理后台 | 已登录且角色允许时，从访客端跳转到既有管理区 |

### 7.2 与既有 Spec 关系

- 承接 `blog-auth-rbac` Non-Goals 中「用户资料页复杂编辑」与前端统一登录体验的补强；**不**重做注册协议与 RBAC 规则
- 与 `blog-media-upload`：头像 URL 可复用上传返回值；资料页与访客会话不在 media Spec 范围
- 与 `blog-auth-refresh`：本期登出不要求服务端 Refresh 失效；若后续落地 refresh，再对齐「真登出」
- 与 `blog-comment-like` / `blog-comment-thread`：评论展示作者公开资料；写评论仍需登录，引导回跳由本期补齐
- 与 `blog-rate-limit`：不在本期做登录限流

### 7.3 现状基线（开 Spec 时）

| 能力 | 现状 |
| --- | --- |
| User 字段 | `id`、`username`、`passwordHash`、`role`、`createdAt`；无展示名/简介/头像 |
| 注册 / 登录 API | `POST /api/auth/register`、`POST /api/auth/login` 已存在 |
| 前端登录 | `/login` 与 `/admin/login` 共用管理登录页；成功默认进 `/admin/articles` |
| 注册页 | 公开布局下 `/register` 已有 |
| 访客 Header | 仅站点导航 + 站点作者头像链到关于页；无账号会话入口 |
| 登出 | 仅管理布局有退出；访客端无 |
| 评论作者展示 | 仅 `username`；文章作者为 `authorName`（现为 username） |

### 7.4 建议默认值（WHAT 级；Plan 可微调并锁定）

| 项 | 建议 |
| --- | --- |
| 展示名长度 | 1–32（可空则展示回退 username） |
| 简介长度 | ≤ 200 |
| 头像 URL 长度 | ≤ 512 |
| 本人资料路径 | Plan 固定（如 `GET/PUT /api/me` 或 `/api/users/me`） |
| 访客登录默认落地 | `/` 或来源公开页 |
| 切换账号 | `clearAuth` + 跳转登录页（清空表单） |
