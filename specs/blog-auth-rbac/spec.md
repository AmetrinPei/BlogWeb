## 1. Problem Statement

MVP 仅有单一管理员登录，无法注册普通用户，也没有作者角色与「仅能改自己的文章」约束，无法支撑多作者博客场景。

本期交付：用户注册与登录（JWT）；角色为访客（未登录）、作者、管理员；文章归属作者；作者仅可修改/删除自己的文章，管理员可管理全部。

---

## 2. Success Metrics

| 指标 | 目标值 | 度量方式 |
| --- | --- | --- |
| 注册登录 | 新用户可注册并登录获得 JWT | 端到端注册→登录→带 Token 请求 |
| 角色区分 | AUTHOR 与 ADMIN 权限不同 | 作者改他人文章返回 403；管理员可改 |
| 文章归属 | 新建文章绑定当前用户为作者 | 创建后响应含 authorId/username |
| 兼容管理员 | 原 admin 账号仍可登录管理端 | 使用种子账号验证 |

---

## 3. User Stories

- 作为访客，我希望注册账号并登录，以便日后评论与创作
- 作为作者，我希望创建并编辑自己的文章，以便发布内容
- 作为作者，我不能修改他人文章，以便保护内容归属
- 作为管理员，我希望管理所有文章与用户相关后台能力，以便运营站点

---

## 4. Acceptance Criteria

- [x] AC-1: 提供注册接口；用户名唯一；密码安全存储（BCrypt）；默认角色 AUTHOR
- [x] AC-2: 登录接口对 ADMIN/AUTHOR 均可用，返回 JWT（含角色信息或可解析角色）
- [x] AC-3: 文章具备 `author` 归属；创建时自动设为当前登录用户
- [x] AC-4: AUTHOR 仅可查看/更新/删除自己的文章（管理列表与详情均隔离）；越权返回 403
- [x] AC-5: ADMIN 可查看/更新/删除任意文章；可访问全部 `/api/admin/**`
- [x] AC-6: 未登录访问需认证接口返回 401
- [x] AC-7: 前端提供注册页与统一登录；管理路由仅 ADMIN 可进入（AUTHOR 可有创作入口或共用受限后台，见 plan）
- [x] AC-8: 种子管理员角色为 ADMIN，行为与 MVP 兼容

---

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| 刷新令牌 / Token 黑名单 | 进阶 |
| 按钮级 RBAC、权限委托 | 超出标准版 |
| 邮箱验证、OAuth | 非本期 |
| 用户资料页复杂编辑 | 仅用户名等最小字段 |

---

## 6. Constraints

- 遵守 constitution.md；密码禁止明文日志
- 权限必须在 Service 层校验
- 兼容现有 `/api/admin/auth/login` 或提供等价登录并保持管理端可用
