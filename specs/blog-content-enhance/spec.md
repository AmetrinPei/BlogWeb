> 依赖：specs/personal-blog（MVP Implemented）  
> 最后更新：2026-07-14

---

## 1. Problem Statement

MVP 文章以纯文本直接发布，缺少草稿/下架状态，也无法配置封面、摘要、阅读量与置顶推荐，难以支撑接近真实博客的编辑与展示体验。

本期交付**内容增强**：正文按 Markdown 管理；文章具备草稿 / 已发布 / 已下架三态；支持封面 URL、摘要、阅读量、置顶与推荐。

---

## 2. Success Metrics

| 指标 | 目标值 | 度量方式 |
| --- | --- | --- |
| 状态隔离 | 公开列表/详情仅出现「已发布」文章 | 准备草稿、已发布、已下架各 ≥1 篇验证 |
| Markdown 可读 | 详情页能渲染常见 MD 语法且脚本被消毒 | 含标题/列表/代码块/危险 script 的样本走查 |
| 阅读量 | 每次成功打开已发布详情，阅读量 +1 | 连续请求详情 3 次，计数递增 |
| 管理字段完整 | 管理端可编辑状态、封面、摘要、置顶、推荐 | CRUD 用例覆盖新字段 |
| 置顶排序 | 公开列表置顶文章排在非置顶之前 | 混合数据列表前 N 条含全部置顶 |

---

## 3. User Stories

- 作为管理员，我希望将文章存为草稿或下架，以便控制公开可见性
- 作为管理员，我希望用 Markdown 编写正文，以便结构化排版
- 作为管理员，我希望设置封面 URL、摘要、置顶与推荐，以便首页与列表更好展示
- 作为访客，我希望看到已渲染的 Markdown 正文与阅读量，以便阅读与判断热度
- 作为访客，我不应看到草稿或已下架文章

---

## 4. Acceptance Criteria

- [x] AC-1: 文章状态为 `DRAFT` / `PUBLISHED` / `OFFLINE` 三选一；新建默认可为草稿或按管理端选择
- [x] AC-2: 公开列表与详情仅返回 `PUBLISHED`；草稿与已下架返回不存在或不可见（详情 404）
- [x] AC-3: 管理端可查看与修改全部状态的文章；可设置 `coverUrl`、`summary`、`pinned`、`recommended`
- [x] AC-4: 正文按 Markdown 存储；访客详情展示渲染后的安全 HTML（危险标签/脚本不可执行）
- [x] AC-5: 公开详情成功访问时 `viewCount` 原子递增 1；列表与详情响应包含 `viewCount`
- [x] AC-6: 公开列表排序：先 `pinned` 降序，再 `publishedAt` 降序
- [x] AC-7: 列表摘要优先使用 `summary`；为空时可截取正文前若干字符（实现细节见 plan）
- [x] AC-8: 既有 MVP 字段（标题、分类、标签、发布时间）行为保持兼容

---

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| 富文本编辑器双轨 | 本期仅 Markdown |
| 图片本地/OSS 上传 | 进阶版；封面仅 URL |
| Redis 累加阅读量 | 进阶版；本期直接写库 |
| 定时发布任务 | 可用 publishedAt + 状态手动控制 |
| 多作者归属与 RBAC | 见 blog-auth-rbac |

---

## 6. Constraints

- 必须兼容现有公开/管理文章 API 路径风格与统一响应格式
- 必须遵守 `constitution.md`
- 公开可见性以「状态 = PUBLISHED」为准（取代 MVP 仅用 `publishedAt <= now` 的隐含草稿语义；未来发布时间的已发布文章仍可不展示，见 plan）
- 禁止引入对象存储或 Redis
