# Feature: 友链与关于页可配

> 状态：Implemented  
> 关联：docx/个人博客-应用场景需求与进阶开发.md §九 `blog-friend-links`；constitution.md  
> 依赖：specs/blog-site-experience（站点配置单例、公开/管理读写）；specs/blog-theme-switch（关于页头像 URL 与视觉呈现已交付，本期不重做）  
> 最后更新：2026-07-15

---

## 1. Problem Statement

`blog-site-experience` 已交付站名、简介、`aboutText` 与**社交链接**（作者本人账号），`blog-theme-switch` 已交付关于页头像 URL 与框饰呈现，但：

- **无友链（blogroll）**：不能在后台维护「友情博客 / 推荐站点」列表；社交链接与友链语义不同，不能混用。
- **关于页仍半静态**：标题、亮点列表、展示名等仍写在前端 `site.js`；改文案需发版，与「站点可后台维护」目标不一致。

本期交付 **ADMIN 可维护的友链列表 + 关于页结构化内容可配并前台展示**：

1. ADMIN 可对友链做增删改、排序；公开只读下发；访客端可见。
2. 关于页关键文案（标题、正文/简介、亮点等）可由 ADMIN 配置；公开接口下发；前台关于页读取配置（缺省有降级默认值）。
3. 不破坏既有 `siteName` / `tagline` / `aboutText` / `socialLinks` / 主题视觉字段行为；**社交链接与友链分字段、分语义**。

不做访客申请友链、审核工作流、友链兑换、外站爬虫探活。

---

## 2. Success Metrics

| 指标 | 目标值 | 度量方式 |
| --- | --- | --- |
| 友链可维护 | ADMIN 可新增、编辑、删除友链，并调整展示顺序 | 管理端操作 → 公开接口顺序与内容一致 |
| 友链前台可见 | 访客未登录即可看到已配置友链（按排序） | 无 Token 请求公开数据 + 前台页面渲染 |
| 关于可配 | ADMIN 修改关于页结构化内容后，前台关于页反映新值 | 管理保存 → 公开配置 → `/about` 展示 |
| 语义不混淆 | `socialLinks` 与友链列表独立；改友链不影响社交链接 | 分别断言两套字段 |
| 权限正确 | 非 ADMIN 不可写；未登录写操作 401 | API 断言 401 / 业务拒绝 |
| 空态可接受 | 无友链或关于字段为空时前台不报错，有合理降级/隐藏 | 清空配置后刷新关于页 |

## 3. User Stories

- 作为站点管理员，我希望维护一份友情链接列表（名称、地址、顺序），以便推荐友站而无需改代码
- 作为站点管理员，我希望编辑关于页的标题、简介与亮点，以便自我介绍可后台更新
- 作为访客，我希望在公开页面看到友链并点击跳转外站，以便发现相关博客
- 作为访客，我希望关于页展示与后台配置一致的内容，以便了解站点作者
- 作为未登录访客，我仍能阅读关于页与友链，以便无需账号即可浏览
- 作为系统运营者，我不希望访客能擅自提交友链申请占用审核人力，以便个人博客保持轻量

## 4. Acceptance Criteria

### 4.1 友链（ADMIN 维护 + 公开只读）

- [x] AC-1: 提供 **ADMIN 可维护友链** 的能力（独立 CRUD 接口，或纳入站点配置更新的一部分；由 Plan 锁定形态）。至少可维护每条友链的：
  - **名称**（必填）
  - **URL**（必填，合法 http/https）
  - **排序值**（必填或可推导；决定前台展示顺序）
  - **可选简介**（可空；有长度上限，Plan 锁定）
  - **可选是否启用**（若本期不做启用开关，则「删除」即下线；Plan 须明确一种）
- [x] AC-2: 友链列表按 **排序字段升序**（同序时次序稳定，规则由 Plan 锁定）返回；公开侧只返回应对访客展示的条目（若有启用字段则仅启用项）
- [x] AC-3: 提供 **公开只读** 获取友链的途径（独立公开接口，和/或公开站点配置响应中的字段；Plan 锁定）。**无需登录**；统一 JSON `{ code, message, data }`
- [x] AC-4: ADMIN 写操作：未登录 **401**；非 ADMIN **拒绝**（403 或等价业务错误）；权限在 **Service 层**强制
- [x] AC-5: 校验：名称/URL 为空或超长、非法 URL scheme（非 http/https）、条数超过约定上限时返回明确业务错误（非 5xx）；上限由 Plan 锁定（建议默认 ≤ 50）
- [x] AC-6: **语义隔离**：友链与既有 `socialLinks` **不得**共用同一存储字段或互相覆盖；公开响应中二者可并存且含义不同

### 4.2 关于页内容可配

- [x] AC-7: 扩展可配置的关于页内容，使 ADMIN 至少可维护并公开下发：

  | 能力 | 要求 |
  | --- | --- |
  | 关于标题 | 可配；空则前台用明确默认文案（如「关于我」） |
  | 关于正文/简介 | 复用或对齐既有 `aboutText`；可配；空则前台隐藏或显示默认简介（行为明确） |
  | 亮点列表 | 可配的字符串列表（有条数与单条长度上限）；空则前台不渲染亮点区 |

  可选字段（Plan 可纳入或明确不做）：展示名（与硬编码 `author` 解耦）、友链区标题文案。

- [x] AC-8: **公开站点（或关于）接口**返回 AC-7 字段；**不破坏**既有 `siteName` / `tagline` / `aboutText` / `socialLinks` 及 `blog-theme-switch` 视觉字段
- [x] AC-9: **ADMIN 更新接口**可写入 AC-7 字段；校验长度与列表规模；未登录 401；非 ADMIN 不可写；权限在 Service 层强制
- [x] AC-10: 访客端 **关于页**（`/about`）读取公开配置渲染标题、简介、亮点；配置缺失时回落前端默认，**不得白屏或抛未捕获错误**
- [x] AC-11: 关于页头像继续使用既有 `aboutAvatarUrl` / 默认资源与框饰（`blog-theme-switch`）；本期**不**要求重做头像视觉

### 4.3 前台展示与管理入口

- [x] AC-12: 访客端至少在一处公开页面展示友链列表（**关于页内区块**，和/或独立「友链」路由；Plan 锁定主入口）。展示名称与可点击 URL；有简介则展示；按排序；无友链时隐藏整块或显示空态文案（二选一，Plan 锁定）
- [x] AC-13: 外链使用安全跳转惯例：`target="_blank"` 且 `rel` 含 `noopener noreferrer`（或等价）
- [x] AC-14: 管理端提供友链编辑入口（站点设置页内区块，或独立管理页；Plan 锁定）；支持调整顺序并保存；保存成功有反馈
- [x] AC-15: 管理端提供关于页结构化字段编辑入口（可与现有「关于文案」表单项合并升级）；保存后公开侧可读

### 4.4 验收与范围边界

- [x] AC-16: 关键路径具备可自动化验收手段（测试或脚本）：至少覆盖「ADMIN 写入友链后公开可读且顺序正确」「非 ADMIN 不可写」「关于字段更新后公开可读」「空友链不 5xx」
- [x] AC-17: 不引入 Elasticsearch、Redis、消息队列、对象存储 SDK、SSR；不新增友链申请/审核表流程

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| 访客提交友链申请 + 审核工作流 | 卡片明确另开；个人站维护成本高 |
| 友链自动探活、死链检测、抓取对方站点信息 | 运维与外呼复杂度超出本期 |
| 把 `socialLinks` 改名为友链或合并两套数据 | 语义不同：本人社交 vs 友站推荐 |
| 重做主题切换、自定义背景、头像框饰 | 已由 `blog-theme-switch` 交付 |
| 富文本/Markdown 关于页（任意 HTML） | XSS 面与消毒成本；本期纯文本/字符串列表足够 |
| Newsletter、SEO、RSS 变更 | 无关或已有独立 Spec |
| 多语言关于页、多作者关于页 | 超出个人博客本期范围 |

## 6. Constraints

- 遵守 `constitution.md`：统一响应 `{ code, message, data }`；业务校验在 **Service 层**；不引入禁止技术栈
- 存储形态二选一或组合，由 **Plan** 锁定：**扩展站点配置单例**（如 JSON 列）**或独立友链表**；关于结构化字段优先扩展站点单例，避免无必要新域
- 后端 domain 归属：友链与关于配置落在既有 `site` 包（或 Plan 证明的最小新包），保持 Controller → Service → Repository
- URL 类字段做长度与 scheme 校验；前台链接属性防 `javascript:` 等危险 scheme；**禁止**把用户输入当可执行脚本插入页面
- 兼容现有 `AboutView` / 站点设置管理页信息架构；允许增区块或路由，不得去掉关于页主导航入口
- Spec 写 WHAT；表结构、接口路径、排序同分规则、条数上限、前台挂载位置由 **Plan** 锁定
- 关键路径验收与 PR 说明引用 `blog-friend-links` 与 Task 编号

## 7. 附录

### 7.1 术语表

| 术语 | 说明 |
| --- | --- |
| 友链 / blogroll | 站点推荐的外部友情博客或站点列表；非作者本人社交账号 |
| 社交链接 `socialLinks` | 作者本人的 GitHub / 邮箱等；已由 `blog-site-experience` 交付 |
| 关于页结构化内容 | 标题、简介（`aboutText`）、亮点列表等组成关于页主体文案的可配字段 |
| 排序字段 | 决定友链前台展示先后的整型或等价序值 |
| 降级默认值 | 公开配置缺失时前端/后端使用的本地默认文案或空列表 |

### 7.2 与既有 Spec 关系

- 承接需求文档 §九 `blog-friend-links` 卡片；补齐 `blog-theme-switch` Non-Goals 中「友链模块、关于页文案结构化重构」
- 扩展 `blog-site-experience` 站点配置能力；**不**弱化其 `socialLinks` 契约
- 关于页头像 URL / 框饰继续遵循 `blog-theme-switch`；本期只补文案与友链
- 与 `blog-user-profile`：用户资料展示名/头像 ≠ 站点关于页展示；勿混用字段语义
- 友链申请审核若未来需要，另开 Spec，不在本期表结构预埋复杂状态机（简单 `enabled` 开关除外）

### 7.3 现状基线（开 Spec 时）

| 能力 | 现状 |
| --- | --- |
| 友链 | 无独立模型/接口/前台区块 |
| 社交链接 | 站点配置 `socialLinks`；管理端可编；关于页底部展示为 socials |
| 关于正文 | 后端 `aboutText` + 管理端「关于文案」；亦作首页副文案来源之一 |
| 关于标题 / 亮点 / 展示名 | 前端 `frontend/src/config/site.js` 写死；`AboutView` 读取 `site.about.*` / `site.author` |
| 关于头像 | `aboutAvatarUrl` + `AboutAvatar` 框饰（theme-switch 已交付） |
| 公开站点接口 | 含 siteName / tagline / aboutText / socialLinks / 主题视觉字段；无 friendLinks / aboutTitle / highlights |

### 7.4 Plan 已锁定（摘要）

| 项 | 锁定值 |
| --- | --- |
| 存储 | `site_settings` 单例 JSON 列 `friend_links`；关于字段同单例 |
| 接口 | 扩展既有 `GET /api/site`、`PUT /api/admin/site`（无独立 CRUD） |
| 友链字段 | `name` / `url` / `description` / `sortOrder`；无启用开关 |
| 上限 | 友链 ≤50；亮点 ≤10；URL 仅 http(s) |
| 未传策略 | 新字段 `null` 保持原值；管理端全量提交 |
| 前台入口 | 仅 `/about` 内「友情链接」区块；空则隐藏；无独立路由 |
| 关于字段 | `aboutTitle` / `aboutDisplayName` / `aboutHighlights` + 既有 `aboutText` |
| 验收 | `FriendLinksSiteTests` + `scripts/acceptance-friend-links.mjs` |

详见 `plan.md`。
