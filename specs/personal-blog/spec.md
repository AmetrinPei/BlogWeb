# Feature: 个人博客（基础版 MVP）

> 状态：Approved  
> 关联：docx/个人博客-应用场景需求与进阶开发.md  
> 最后更新：2026-07-13

---

## 1. Problem Statement

需要一套可长期维护的个人内容站点，用于发布技术笔记与作品展示。

当前缺少统一平台，内容分散、难以按分类检索，也无法通过后台集中管理文章。

本期交付**基础版 MVP**：访客可浏览与检索文章，管理员可登录并完成内容与分类/标签管理。

## 3. User Stories

### 访客

- 作为访客，我希望浏览文章列表并翻页，以便按时间阅读内容
- 作为访客，我希望查看文章详情（标题、正文、发布时间），以便完整阅读
- 作为访客，我希望按分类或标签筛选文章，以便聚焦某一主题
- 作为访客，我希望按标题、正文关键词搜索，以便快速定位文章

### 管理员

- 作为管理员，我希望登录后台，以便管理站点内容
- 作为管理员，我希望对文章增删改查，以便维护发布内容
- 作为管理员，我希望管理分类与标签，以便组织文章结构

---

## 4. Acceptance Criteria

### 4.1 访客端

- [x] AC-1: 文章列表按发布时间倒序展示，支持分页（默认每页 10 条）
- [x] AC-2: 文章详情展示标题、正文、发布时间；不存在时返回明确错误
- [x] AC-3: 可按单一分类或单一标签筛选，结果仅为已发布文章
- [x] AC-4: 标题关键词搜索支持部分匹配，无结果时返回空列表（不报错）
- [x] AC-5: 列表与详情仅展示已发布文章，不暴露草稿或已删除内容

### 4.2 管理端

- [x] AC-6: 管理员凭账号密码登录，成功后获得可访问管理接口的凭证
- [x] AC-7: 文章支持创建、查看、更新、删除；字段含标题、正文、分类、标签、发布时间
- [x] AC-8: 分类支持增删改查；删除已被文章引用的分类时给出明确提示
- [x] AC-9: 标签支持增删改查；文章可关联多个标签
- [x] AC-10: 未登录或凭证无效时，所有管理接口返回未授权错误

### 4.3 通用

- [x] AC-11: 前后端通过 REST API + JSON 交互
- [x] AC-12: 接口响应统一为 `code / message / data` 结构
- [x] AC-13: 非法参数与业务异常返回可读错误信息，不暴露堆栈

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| 用户注册、多角色权限 | 本期仅单一管理员，留待标准版 |
| 评论、点赞、互动 | 超出 MVP 范围 |
| Markdown/富文本编辑器 | 本期正文为纯文本或简单格式即可 |
| 草稿 / 发布 / 下架状态机 | 本期视为直接发布，留待标准版 |
| 封面图、摘要、阅读量、置顶 | 展示增强，非 MVP 必需 |
| 全文搜索、Elasticsearch | 本期仅标题关键词搜索 |
| 图片上传、媒体管理 | 留待进阶版 |
| SSR、SEO、sitemap | 留待进阶版 |
| Redis 缓存、限流、监控 | 留待进阶版 |

> 增删说明：直接在表格中增删行，并同步更新第 4 节 AC 编号。

## 6. Constraints

- 必须采用前后端分离架构，通过 REST API 通信
- 必须提供访客端（阅读）与管理端（后台）两类界面或入口
- 必须持久化存储文章、分类、标签及关联关系
- 管理端操作必须经身份认证保护
- 建议技术栈：Spring Boot + Vue（实现细节见 `plan.md`，可替换）
