# Feature: 评论与点赞（标准版）

> 状态：Implemented  
> 关联：docx/个人博客-应用场景需求与进阶开发.md §2 标准版·互动  
> 依赖：blog-auth-rbac（需登录用户）  
> 最后更新：2026-07-14

---

## 1. Problem Statement

MVP 与内容增强阶段缺少读者互动，文章无法评论与点赞，互动闭环不完整。

本期交付：已发布文章的一级评论（不做楼中楼）；文章与评论的点赞（登录用户，幂等）。

---

## 2. Success Metrics

| 指标 | 目标值 | 度量方式 |
| --- | --- | --- |
| 一级评论 | 登录用户可评论已发布文章；列表按时间正序或倒序稳定 | API 用例 |
| 未登录拦截 | 未登录发评论/点赞返回 401 | 无 Token 调用 |
| 点赞幂等 | 同一用户对同一目标重复点赞不重复计数；可取消 | 连续 toggle |
| 仅一级 | 不存在 parentId 回复结构 | 数据模型与 API 无回复字段 |

---

## 3. User Stories

- 作为登录用户，我希望评论文章，以便交流
- 作为登录用户，我希望对文章或评论点赞/取消，以便表达态度
- 作为访客，我希望查看评论列表与点赞数，以便参考互动

---

## 4. Acceptance Criteria

- [x] AC-1: 登录用户可对已发布文章发表一级评论；内容非空有长度上限
- [x] AC-2: 公开可分页或列表获取某文章评论（含作者用户名、时间、点赞数）
- [x] AC-3: 评论作者或管理员可删除自己的评论（ADMIN 可删任意）
- [x] AC-4: 登录用户可对文章或评论点赞；再次请求取消点赞；计数正确
- [x] AC-5: 未登录写操作 401；对草稿/下架文章不可评论（404 或业务错误）
- [x] AC-6: 无楼中楼/回复接口

---

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| 楼中楼、评论审核流 | 标准版仅一级 |
| 敏感词、验证码 | 进阶 |
| 通知系统 | 非本期 |

---

## 6. Constraints

- 遵守 constitution.md；点赞需唯一约束防并发双插
- 依赖 JWT 登录用户
