# Feature: 阅读体验打磨

> 状态：Implemented  
> 关联：docx/个人博客-应用场景需求与进阶开发.md §九 `blog-reading-ux`；constitution.md  
> 依赖：specs/personal-blog（MVP）；specs/blog-content-enhance（Markdown 正文、公开详情、XSS 消毒）  
> 最后更新：2026-07-14

---

## 1. Problem Statement

标准版详情页已能渲染 Markdown 并消毒危险 HTML，但长文缺少可导航目录，代码块仅为朴素 `<pre>`，列表/详情加载仅「加载中…」文案，搜索无结果也缺少可区分的空态。读者在长文中定位章节、扫读代码与等待加载时体验偏弱。

本期交付**阅读体验打磨**（以前端局部组件为主）：详情页由正文多级标题生成可点击 TOC（锚点跳转）；代码块语法高亮且不拖垮首屏（允许延迟加载高亮库）；文章列表与详情具备骨架或占位加载态；带关键词搜索无结果时有明确空态。不新增公开 API，不改文章状态机与权限。

---

## 2. Success Metrics

| 指标 | 目标值 | 度量方式 |
| --- | --- | --- |
| TOC 可用 | 含 h2/h3（或等价多级标题）的长文可生成可点击目录，点击滚动到对应标题 | 准备 ≥1 篇含 ≥3 个标题的样本，手工/脚本断言 TOC 项数与锚点存在 |
| XSS 不回退 | 详情渲染后危险 HTML（如 `<script>`、事件属性）仍不可执行 | 沿用 content-enhance 消毒样本；TOC/高亮改造后回归 |
| 首屏不拖垮 | 高亮库不阻塞首屏关键内容展示；无代码块时可不加载高亮资源 | 无代码块文章 Network 无高亮 chunk；有代码块时正文先可见再高亮 |
| 加载可感知 | 列表、详情在请求未完成时展示骨架或占位，而非仅一行「加载中…」 | 限速或 Mock 延迟下走查 |
| 搜索空态 | 有 keyword 且无命中时展示可区分的空态（含提示与清空/返回动作） | 搜索不可能命中的词，断言空态文案与操作入口 |

---

## 3. User Stories

- 作为访客，我希望在长文侧边或文首看到目录，以便快速跳到某一章节
- 作为访客，我希望代码块有语法高亮，且打开文章时不要长时间白屏
- 作为访客，我希望列表与详情加载时看到骨架占位，以便感知进度
- 作为访客，我希望搜索无结果时看到明确空态，并能一键清空条件继续浏览
- 作为访客，我不应因 TOC 或高亮改造而执行文章中的恶意脚本

## 4. Acceptance Criteria

- [x] AC-1: 公开文章详情页根据正文中的多级标题（至少覆盖 Markdown 渲染后的 `h2`/`h3`）生成**可点击 TOC**；点击某项后页面滚动到对应标题位置（锚点或等价平滑滚动）
- [x] AC-2: 每个进入 TOC 的标题在正文中具备稳定可定位的 `id`（或等价锚点）；同一文章内锚点不冲突（重名标题有去重策略，见 Plan）
- [x] AC-3: 正文无可用标题时，**不展示空 TOC 壳**（隐藏目录区域或等价）
- [x] AC-4: 代码块具备语法高亮（语言可从 Markdown fence 推断；未知语言降级为纯文本高亮或原样）；**高亮实现不得阻塞首屏关键内容**（正文 HTML 可先展示，高亮库允许异步/按需加载）
- [x] AC-5: 改造后详情页仍对渲染 HTML 做 XSS 消毒；含 `<script>`、`onerror=` 等危险内容的样本**不可执行**（与 `blog-content-enhance` AC-4 同级要求，不得回退）
- [x] AC-6: 公开**文章列表**在加载中展示骨架或卡片占位（不得仅保留单行「加载中…」作为唯一反馈）
- [x] AC-7: 公开**文章详情**在加载中展示骨架或内容占位（标题/正文区可见占位结构）
- [x] AC-8: 公开文章列表在**存在非空 keyword 且结果为空**时展示搜索无结果空态：说明无匹配，并提供至少一种恢复操作（清空关键词 / 清空全部筛选 / 返回全部文章）
- [x] AC-9: 无 keyword 的空列表（如分类下无文）可沿用通用空态，但文案或样式须与「搜索无结果」可区分，或共用组件但文案随场景变化
- [x] AC-10: **不新增**公开 REST API；不修改文章状态机、角色权限与公开可见性规则
- [x] AC-11: 关键体验路径具备可验收手段：至少手工走查清单写入 Plan/Tasks；XSS 与 TOC 锚点优先自动化或半自动脚本（可选）

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| SSR / 预渲染 / Nuxt | constitution 禁止；留给 `blog-seo` |
| 改文章状态机、RBAC、公开列表/详情 API 契约 | 卡片 Non-Goals；Constraints 明确前端为主 |
| 服务端生成 TOC 字段或独立 TOC API | 本期客户端从正文生成即可 |
| 全文阅读进度条、预计阅读时长、字体大小设置 | 超出本期 |
| 暗色主题 / 多主题 | 见 `blog-theme-switch` |
| 搜索结果关键词高亮片段 | 属 `blog-search-enhance` Non-Goals 后续项 |
| 评论区骨架以外的互动改版 | 评论加载可保持简单文案；楼中楼见 `blog-comment-thread` |
| 管理端 Markdown 预览 TOC | 本期仅访客公开阅读页 |

---

## 6. Constraints

- 必须遵守 `constitution.md`：不引入 SSR；Markdown→HTML 必须 XSS 消毒；不引入 Elasticsearch / Redis
- **以前端局部组件为主**；若实现中发现必须新增公开 API，须先修订本 Spec 并升格评审后再改后端
- 复用现有 `marked` + `DOMPurify` 消毒链路；TOC 锚点注入不得绕过消毒或重新引入未消毒 HTML
- 语法高亮库若新增 npm 依赖，须在 Plan 中选型说明（体积、按需加载、许可证）；禁止为高亮引入 SSR 框架
- 访客端视觉须融入现有治愈系设计系统（`style.css` 变量），避免另起一套卡片堆砌风格
- 关键路径验收与 PR 说明引用 `blog-reading-ux` 与 Task 编号

## 7. 附录

### 7.1 术语表

| 术语 | 说明 |
| --- | --- |
| TOC | Table of Contents，由正文标题生成的文章目录 |
| 锚点 | 标题上的 `id` 或等价定位，供 TOC 点击跳转 |
| 骨架 / 占位 | 加载中展示的灰块或结构占位，模拟最终布局轮廓 |
| 搜索无结果空态 | 有关键词过滤且命中数为 0 时的专用空状态 UI |
| 首屏关键内容 | 标题、元信息与正文首段等，不等待高亮库即可阅读 |

### 7.2 与既有 Spec 关系

- 建立在 `blog-content-enhance` 的 Markdown 渲染与 XSS 消毒之上，**不得削弱**其安全 AC
- 搜索无结果空态配合 `blog-search-enhance` 的空列表 API 行为（`code=0`、空 items），本期只补前端呈现
- 与 `blog-seo`（SSR/sitemap）隔离：本期纯 CSR 体验

### 7.3 现状基线（开 Spec 时）

| 能力 | 现状 |
| --- | --- |
| MD 渲染 | `ArticleDetailView`：`marked` + `DOMPurify.sanitize` |
| TOC | 无 |
| 代码高亮 | 仅 CSS 美化 `<pre>`，无语法着色库 |
| 列表/详情加载 | 文案「加载中…」 |
| 列表空态 | 统一「没有找到相关文章。」，未区分搜索 |
