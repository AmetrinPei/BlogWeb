# Feature: 主题切换与访客端视觉氛围

> 状态：Implemented  
> 关联：docx/个人博客-应用场景需求与进阶开发.md §九 `blog-theme-switch`；constitution.md  
> 依赖：specs/blog-site-experience（站点配置、访客壳层）；建议复用 specs/blog-media-upload（自定义背景图 / 头像 / 贴图 URL）

> 预览：`specs/blog-theme-switch/效果预览.html`

## 1. Problem Statement

`blog-site-experience` 已交付站点名、简介与社交链接，访客端有统一治愈系 token，但：

- **无主题切换**：仅单一亮色；无暗色 / 命名预设；刷新无法保持访客偏好。
- **主界面背景偏平**：`body` / 布局底为近纯白，装饰层仅少量低透明度星星与波浪，缺少层次与氛围。
- **「关于我」头像偏素**：固定小圆图 + 细边框，缺少与主题联动的光晕、环饰与装饰语言。
- **首页右侧贴图偏平**：静态单张 `hero-illustration.svg` 色块插画，无分层贴纸感，也不随主题变色或可后台替换。

本期交付 **访客端主题切换 + 视觉氛围增强 + 站点级可自定义背景（及关键视觉资源）**：

1. 访客可切换至少 **亮色 / 暗色**（及可选命名预设），偏好本地持久化，刷新保持；关键对比度可读。
2. 默认主题下，**主界面背景**、**关于我头像呈现**、**首页右侧贴图**须达到本 Spec「视觉设计目标」的可验收观感（分层氛围，而非单色底 + 单张扁图）。
3. **站点所有者（ADMIN）** 可配置界面背景（纯色 / 渐变预设 / 图片 URL）及关于页头像 URL、首页贴图 URL；公开接口下发；前台按主题与自定义配置合成展示。清空自定义后回落到主题默认氛围资源。

不做在线主题商店、访客任意上传全站皮肤、管理端暗色、完整主题编辑器。

## 2. Success Metrics

| 指标 | 目标值 | 度量方式 |
| --- | --- | --- |
| 主题可切换且持久 | 访客可切亮/暗（及约定预设）；刷新后仍为上次选择 | UI 操作 + localStorage（或等价）断言；二次进入一致 |
| 对比度可读 | 亮/暗主题下正文、导航、按钮文字与背景对比可接受（非灰字贴灰底） | 人工抽查首页 / 关于 / 文章列表关键元素 |
| 背景有层次 | 默认主题下主界面背景非单一纯色平铺；存在可感知的分层氛围 | 对照附录 7.4；亮/暗各一眼可区分 |
| 头像与贴图更丰富 | 关于页头像、首页右侧贴图具备分层装饰/环饰或贴纸层，并随主题 token 变色 | 对照附录 7.4；切换主题后色调同步变化 |
| 背景可自定义 | ADMIN 可配置背景模式与资源；保存后访客端反映；非法清空/恢复默认行为明确 | 管理端保存 → 公开配置 → 前台展示；恢复默认后无残留自定义图 |
| 自定义不毁可读性 | 自定义背景图启用时，正文与导航仍可读（须有遮罩或半透明底） | 深色/浅色样图各测一次 |
| 不破坏站点配置 | 既有 siteName / tagline / aboutText / socialLinks 行为不变 | 回归公开站点接口与前台顶栏/页脚 |

## 3. User Stories

- 作为访客，我希望一键切换亮色 / 暗色（或命名主题），以便按环境与喜好阅读
- 作为访客，我希望刷新后仍保持我的主题选择，以便不必每次重选
- 作为访客，我希望主界面背景、关于页头像与首页贴图更有氛围与层次，以便站点不显得「白板 + 扁图」
- 作为站点管理员，我希望配置自定义界面背景（颜色 / 渐变 / 图片），以便品牌化首页与全站壳层
- 作为站点管理员，我希望可替换「关于我」头像与首页右侧贴图资源，以便与背景主题统一
- 作为站点管理员，我希望清除自定义后立刻回到主题默认视觉，以便试错可回退
- 作为未登录访客，我仍能切换主题与看到站点默认/已配置视觉，以便无需账号即可体验

## 4. Acceptance Criteria

### 4.1 主题切换（访客偏好）

- [x] AC-1: 访客端壳层（如顶栏）提供**可见的主题切换控件**；支持 **`light`（亮色）** 与 **`dark`（暗色）**（本期不做第三命名预设；见 Plan）
- [x] AC-2: 选择主题后，**立即**应用到访客端公开区（布局背景、文字、卡片、装饰与贴图色调）；**刷新后保持**（localStorage 键 **`blog.theme`**）
- [x] AC-3: **首次访问**无本地偏好时：使用站点配置中的**默认主题**（见 AC-10）；若站点未配置默认，则回落 `light`
- [x] AC-4: 亮色与暗色下，导航、正文、次要文案、主按钮在对应背景上**保持可读**；不得出现「暗主题仍强制白底大面积刺眼」或「亮主题灰字贴浅灰不可辨」的关键路径问题
- [x] AC-5: 主题切换**仅作用于访客端公开区**；本期**不要求**管理后台（Element Plus）同步暗色

### 4.2 默认视觉增强（主题内置氛围）

- [x] AC-6: **主界面背景（默认）**：在无自定义背景图时，访客端布局背景须为**分层氛围**，不得仅为单一纯色平铺。至少同时满足：
  - 与当前主题绑定的底色 / 柔和渐变（CSS 变量驱动）；
  - 可感知的环境层（如柔光色块 / 光斑 / 点阵或等价装饰），且随 `light` / `dark` 切换改变明度与色相；
  - 装饰不阻挡点击与选中正文（`pointer-events` 不影响交互）
- [x] AC-7: **「关于我」头像（默认呈现）**：关于页头像区须具备**比现网更丰富的视觉框饰**，至少包括：
  - 展示边长 **140px**（Plan 锁定）；
  - 与主题 token 联动的**环饰或光晕**（非仅 1px 实线描边）；
  - 至少一处与全站装饰语言一致的点缀（如角标星点、软阴影层次）；
  - 头像图本身可为站点配置 URL 或默认资源；框饰随主题变色
- [x] AC-8: **首页右侧贴图（默认呈现）**：首页 hero 右侧须为**分层贴图/插画构图**，不得仅为现网式「单矩形底 + 单层扁插画」且无主题联动。至少包括：
  - 主视觉层（插画或图片）；
  - ≥ 1 层浮动点缀（贴纸、色块、星点等），颜色取自主题 CSS 变量；
  - 切换主题时点缀或主视觉色调可感知变化；
  - 窄屏下有明确降级（可收起点缀或改单列），不遮挡标题与 CTA

### 4.3 站点级自定义（ADMIN + 公开下发）

- [x] AC-9: 扩展站点配置（单例），使 ADMIN 可维护且公开可读至少下列字段（具体字段名由 Plan 锁定）：

  | 能力 | 要求 |
  | --- | --- |
  | 默认主题 | 枚举：至少含 `light` / `dark`（及本期启用的预设） |
  | 背景模式 | 枚举：至少 `theme`（跟随主题默认氛围） / `color` / `gradient` / `image` |
  | 背景色 | `color` 模式使用的色值（合法 CSS 色或受限格式，Plan 锁定校验） |
  | 背景渐变 | `gradient` 模式使用的预设 id 或受限渐变描述（Plan 锁定；禁止任意原始 CSS 注入） |
  | 背景图 URL | `image` 模式；可复用 `blog-media-upload` 返回的本站 URL 或既有允许的 http(s) URL |
  | 关于页头像 URL | 可空；空则用前端默认资源 |
  | 首页贴图 URL | 可空；空则用主题默认分层贴图资源 |

- [x] AC-10: **公开站点接口**返回上述主题/视觉字段；前台据此渲染。既有 `siteName` / `tagline` / `aboutText` / `socialLinks` **不得破坏**
- [x] AC-11: **ADMIN 更新接口**可写入 AC-9 字段；非 ADMIN 不可改；未登录写操作 **401**；权限在 **Service 层**强制
- [x] AC-12: **背景合成规则**（行为可测）：
  - `theme`：仅用当前主题的默认分层氛围（AC-6）；
  - `color` / `gradient`：在主题 token 之上应用配置底，仍保留不挡交互的轻量装饰层（可减弱）；
  - `image`：展示背景图（`cover` 类铺满合理区域）+ **必须有可读性遮罩或内容区实底**，保证顶栏与正文可读；
  - 访客本地主题偏好仍决定**前景 token**（文字/卡片/强调色）；自定义背景不取消亮/暗切换
- [x] AC-13: ADMIN 将背景模式设回 `theme`，或清空自定义头像/贴图 URL 后，前台**立即**（刷新公开配置后）回落默认资源，无残留错误图
- [x] AC-14: 管理端「站点设置」提供上述字段的编辑入口；背景图 / 头像 / 贴图支持填写 URL，并**建议**复用已有上传能力选择本站图片；保存成功有反馈
- [x] AC-15: URL 类字段须做长度与基本合法性校验（超长、明显非法 scheme 拒绝）；**禁止**把任意用户输入当作可执行脚本插入页面（属性/CSS 注入防护策略由 Plan 说明）

### 4.4 验收与范围边界

- [x] AC-16: 关键路径具备可自动化或脚本化抽查手段：至少覆盖「主题偏好持久化键存在」「公开站点响应含新字段」「ADMIN 更新后公开可读」「非 ADMIN 不可写」；视觉观感以人工对照附录 7.4 勾选
- [x] AC-17: 不引入 Elasticsearch、Redis、消息队列、对象存储 SDK、SSR；自定义图走既有本地上传或外链 URL

## 5. Non-Goals

| 不做 | 原因 |
| --- | --- |
| 在线主题商店、用户投稿皮肤、CSS 任意粘贴 | 卡片明确不做完整主题商店；任意 CSS 有 XSS/破坏风险 |
| 访客账号级云端同步主题偏好 | 本期 localStorage 即可；多端同步超出个人博客范围 |
| 管理后台 Element Plus 暗色自适应 | 本期仅访客端；可后续另开 |
| 按路由/按文章独立换肤 | 超出「全站主题 + 站点视觉资源」 |
| 视频背景、WebGL、粒子引擎重依赖 | 性能与维护成本；分层 CSS/SVG 足够 |
| 友链模块、关于页文案结构化重构 | 见 `blog-friend-links`；本期只增强头像呈现与资源 URL |
| 重做 RBAC、评论、搜索 | 无关 |
| 强制所有访客跟随 ADMIN 默认主题、禁止本地覆盖 | 默认仅「无偏好时」用站点默认；有本地偏好以访客为准 |

## 6. Constraints

- 遵守 `constitution.md`：访客端自写 CSS（治愈系设计系统）；不引入禁止技术栈
- 统一响应 `{ code, message, data }`；站点配置仍为**单例**扩展字段，避免为视觉另起无关表（除非 Plan 证明 JSON 列必要）
- 自定义图片优先复用 `blog-media-upload` 本地 URL；不因此引入 OSS SDK
- 业务校验（角色、字段长度、枚举合法性）在 **Service 层**；前端校验不可替代
- 主题与背景须兼容现有 `PublicLayout` / `DecorBackground` / `HomeView` / `AboutView` 信息架构，允许替换实现但不得去掉顶栏主导航与关于/首页入口
- 关键路径验收与 PR 说明引用 `blog-theme-switch` 与 Task 编号
- Spec 写 WHAT；token 名、表字段、localStorage key、具体渐变预设列表由 **Plan** 锁定

## 7. 附录

### 7.1 术语表

| 术语 | 说明 |
| --- | --- |
| 主题（theme mode） | 控制 CSS 变量集的命名模式，如 `light` / `dark` / 命名预设 |
| 访客偏好 | 浏览器本地保存的主题选择；优先于「仅展示」，次于不存在时的站点默认 |
| 站点默认主题 | ADMIN 配置的 `defaultTheme`；仅当访客无本地偏好时生效 |
| 背景模式 | `theme` / `color` / `gradient` / `image`，决定壳层底如何绘制 |
| 分层氛围 | 底色/渐变 + 环境装饰层（光斑/点阵等），非单色 `background` |
| 首页贴图 | 首页 hero 右侧主视觉 + 点缀层的合称（可配置主视觉 URL） |
| 框饰 | 关于页头像外的环、光晕、阴影与点缀，独立于头像位图本身 |

### 7.2 与既有 Spec 关系

- 承接 `blog-site-experience` Non-Goals 中「多主题切换」
- 扩展站点配置读写，兼容既有字段

- 与 `blog-friend-links`：关于页文案/友链后台化另开；本期可继续用现有 about 文案来源，但头像 URL 纳入站点视觉配置

### 7.3 现状基线（开 Spec 时）

| 能力 | 现状 |
| --- | --- |
| 主题切换 | 无；`:root` 固定亮色 token |
| 主界面背景 | `--bg: #ffffff` + `DecorBackground` 三星一点波纹，整体偏平 |
| 关于页头像 | 前端写死 `/avatar.svg`，96px 圆图 + 细高亮边 |

### 7.4 视觉设计目标（验收对照，非实现绑死）

> 实现可用 CSS 变量 + SVG/组件重构；下列为**观感验收清单**，Plan 可细化组件拆分。

#### A. 主界面背景

| 要求 | 说明 |
| --- | --- |
| 拒绝纯平 | 禁止「整页只有一个实心 `--bg`、几乎无第二层」作为最终态 |
| 亮色氛围 | 浅色底 + 低饱和薄荷/桃/丁香柔光或渐变洗色；装饰透明度提高至可感知但仍克制 |
| 暗色氛围 | 深色底（非死黑一块）+ 降低亮度的同色系光斑；卡片与正文区有明确浮起层次 |
| 自定义图 | 图上须有半透明遮罩或内容容器实底；顶栏文字对比达标 |
| 动效预算 | 建议 1～2 处极慢呼吸/漂移（装饰层）；尊重 `prefers-reduced-motion` |

#### B. 「关于我」头像

| 要求 | 说明 |
| --- | --- |
| 尺寸与存在感 | 展示边长建议 ≥ 128px；在卡片上方形成视觉锚点 |
| 环饰 | 双环或渐变环（取 `--primary` / `--accent-*` / `--highlight`），随主题变 |
| 光晕 | 头像后有柔和光斑或扩散阴影，暗色下改为低亮度辉光 |
| 点缀 | 1～3 个小装饰（星、点）环绕，与全站 Decor 语言一致 |
| 自定义图 | 配置 URL 后替换位图；框饰仍在 |

#### C. 首页右侧贴图

| 要求 | 说明 |
| --- | --- |
| 分层 | 主视觉 + ≥ 1 浮动贴纸/色块层；避免单层「贴一张扁 SVG」 |
| 主题联动 | 贴纸描边/填充用 CSS 变量；切暗色后明显换调 |
| 可替换 | ADMIN 可换主视觉图；点缀层可保留在图之上或旁侧 |
| 动效 | 建议贴纸层轻微浮动（幅度小）；`prefers-reduced-motion` 时静止 |
| 响应式 | ≤860px 可改为标题上、贴图下或缩小点缀，不压住 CTA |

#### D. 设计方向约束（对齐项目前端规则）

- 保持现有**治愈系**方向（薄荷主色 + 桃/丁香点缀），扩展为可切换 token，而不是换成另一套无关品牌风
- 避免：紫白渐变套娃、大面积 glow 霓虹、过度圆角胶囊堆砌、仪表盘式多卡片首屏
- 首屏信息结构不变：品牌/欢迎语、一句导语、CTA、右侧主视觉；不把统计条、日程塞进 hero

### 7.5 建议信息架构（供 Plan 裁剪）

```text
访客端
├── 主题切换控件（顶栏）
├── 应用 data-theme / class → CSS 变量
├── 壳层背景 = f(访客主题, 站点 backgroundMode/资源)
├── 首页 hero 右侧 = 默认分层贴图 | 自定义主视觉 URL + 点缀

├── 默认主题
├── 背景：模式 + 色/渐变/图
├── 关于页头像 URL（上传或粘贴）
└── 首页贴图 URL（上传或粘贴）
```
